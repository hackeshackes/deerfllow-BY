### 2.2 关键架构原则(继承战略 spec §5.3 不变量)

| 原则 | v1.6.x 体现 |
|---|---|
| Harness / App 单向依赖 | canvas / collaboration / slack 全在 `app/`,不反向 import `app.*` |
| 协议 + 多实现 | `WorkflowStore` Protocol,默认 InMemory,P1 加 SQLite(同 v1.5.10 multitenancy 风格) |
| 失败显式 | WorkflowExecutor 单节点失败 → 记录到 step.error,workflow 继续(配置项 `fail_fast` 默认 False) |
| 不变量校验 | dataclass(frozen=True) + `__post_init__` 校验,同 v1.5.10 ResourceQuota 模式 |
| 配置可注入 | `MICX_CANVAS_STORE=memory\|sqlite`、`MICX_WORKFLOW_FAIL_FAST=true\|false` |
| 不可变优先 | Workflow / Node / Edge / WorkflowVersion 全 frozen dataclass;更新即新建 |
| Embed 优先(性能) | AGENT 节点走 `deerflow.client` embed,绕开 LangGraph HTTP server,降低 execute 延迟 |

### 2.3 数据流:画布执行一次

```
前端编辑画布 → Canvas.save()
   → PUT /api/workflows/{id}  (body: 完整 Workflow)
   → canvas/routers/workflows.py 校验(workspace_id, DAG 引用合法性)
   → InMemoryWorkflowStore.upsert(workflow)       ← version 自增
   → VersionManager.commit(workflow_id, snapshot)
   → 200 OK + WorkflowResponse

前端点 Run
   → POST /api/workflows/{id}/execute  (body: {inputs, workspace_id})
   → middleware.require_user + require_workspace_member
   → multitenancy.usage_tracker.quota_pre_check(workspace_id, est_tokens)
       超额 → 429 + enforce_mode=hard 阻断 / soft 警告
   → WorkflowExecutor.execute(workflow, inputs)
       拓扑排序 → 按序遍历节点 → 节点 executor.run(inputs)
         AGENT → deerflow.client.DeerFlowClient.chat()
         TOOL  → deerflow.tools.builtins registry
         PROMPT→ jinja-like render
         BRANCH→ AST 评估 true/false 选后续节点
         LOOP  → iterate body,累积 outputs
       每节点产生 ExecutionStep {node_id, started_at, ended_at, status, output, error}
   → WorkflowExecution 落 store
   → 200 OK + {outputs, steps, total_tokens, total_duration_ms}
```

### 2.4 数据流:PublishService 跨部门发布

```
用户在 Thread A(workspace=市场部)详情页点 Publish
   → PublishButton 弹窗(选目标 workspace,例如"销售部")
   → POST /api/threads/{A}/publish  body: {target_workspace_id}
   → middleware.require_user
   → workspace_member(actor, target_workspace_id) AND workspace_member(actor, source_thread.workspace_id)
   → PublishService.publish(source=A, target=B, actor)
       1) 在 B 创建新 Thread T_B
       2) 复制 A 的 messages 到 T_B
       3) T_B.published_from_thread_id = A.published_from_thread_id or A.id  ← 链式保留最初源
       4) A.publish_history.append({T_B.id, by_actor, at})
       5) publish_history 在 Thread 加 maxlen=50 上限(防无限增长)
   → 200 OK + T_B 详情
```

## §3 接口契约

### 3.1 画布后端核心模型

```python
# canvas/models.py
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Any, Mapping

class NodeKind(str, Enum):
    AGENT = "agent"
    TOOL = "tool"
    PROMPT = "prompt"
    BRANCH = "branch"
    LOOP = "loop"

class WorkflowStatus(str, Enum):
    DRAFT = "draft"
    PUBLISHED = "published"
    ARCHIVED = "archived"

@dataclass(frozen=True)
class WorkflowNode:
    id: str
    kind: NodeKind
    config: Mapping[str, Any]      # kind-specific: model/prompt/tool_name/condition/iterations
    position: tuple[float, float]  # 画布坐标

    def __post_init__(self) -> None:
        if self.kind not in NodeKind:
            raise ValueError(f"Unknown NodeKind: {self.kind}")
        # kind-specific 校验
        if self.kind == NodeKind.LOOP:
            n = self.config.get("iterations", 1)
            if not isinstance(n, int) or n < 1 or n > 1000:
                raise ValueError("LOOP iterations must be int in [1, 1000]")
        if self.kind == NodeKind.BRANCH:
            cond = self.config.get("condition")
            if not cond or not isinstance(cond, str):
                raise ValueError("BRANCH requires non-empty 'condition' string")

@dataclass(frozen=True)
class WorkflowEdge:
    id: str
    source_node_id: str
    target_node_id: str
    condition: str | None = None   # BRANCH 节点 true/false 分支标签

@dataclass(frozen=True)
class Workflow:
    id: str
    name: str
    workspace_id: str
    status: WorkflowStatus = WorkflowStatus.DRAFT
    nodes: tuple[WorkflowNode, ...] = ()
    edges: tuple[WorkflowEdge, ...] = ()
    created_at: datetime = field(default_factory=datetime.utcnow)
    updated_at: datetime = field(default_factory=datetime.utcnow)
    version: int = 1

    def __post_init__(self) -> None:
        if not self.name.strip():
            raise ValueError("Workflow name must not be empty")
        if not self.workspace_id.strip():
            raise ValueError("workspace_id is required")
        node_ids = {n.id for n in self.nodes}
        for e in self.edges:
            if e.source_node_id not in node_ids or e.target_node_id not in node_ids:
                raise ValueError(f"Edge {e.id} references unknown node")
```

### 3.2 Store Protocol

```python
# canvas/store.py
from typing import Protocol

class WorkflowStore(Protocol):
    def get(self, workflow_id: str) -> Workflow | None: ...
    def list_by_workspace(self, workspace_id: str) -> list[Workflow]: ...
    def upsert(self, workflow: Workflow) -> Workflow: ...   # 自增 version,改 updated_at
    def delete(self, workflow_id: str) -> None: ...

class VersionStore(Protocol):
    def commit(self, workflow: Workflow) -> WorkflowVersion: ...
    def list_versions(self, workflow_id: str) -> list[WorkflowVersion]: ...
    def get_version(self, workflow_id: str, version: int) -> WorkflowVersion | None: ...

class ExecutionStore(Protocol):
    def save(self, execution: WorkflowExecution) -> None: ...
    def list_for_workflow(self, workflow_id: str, limit: int = 50) -> list[WorkflowExecution]: ...
```

### 3.3 Executor + 节点

```python
# canvas/nodes/base.py
class NodeExecutor(Protocol):
    async def execute(self, config: Mapping[str, Any], inputs: Mapping[str, Any]) -> NodeOutput: ...

@dataclass(frozen=True)
class NodeOutput:
    outputs: Mapping[str, Any]
    error: str | None = None
    metadata: Mapping[str, Any] = field(default_factory=dict)

# canvas/executor.py
class WorkflowExecutor:
    def __init__(
        self,
        store: WorkflowStore,
        node_executors: Mapping[NodeKind, NodeExecutor],
        usage_tracker: InMemoryUsageTracker | None = None,
        fail_fast: bool = False,
    ): ...
    async def execute(self, workflow: Workflow, inputs: Mapping[str, Any]) -> WorkflowExecution: ...

# 节点实现要点
# - agent.py  → deerflow.client.DeerFlowClient().chat(message, thread_id)
# - tool.py   → deerflow.tools.builtins registry,按 config["tool_name"] 查找
# - prompt.py → 自实现轻量 jinja-like {{var}} 插值(避免引入 jinja2 依赖)
# - branch.py → AST evaluator: var op value,op ∈ {==,!=,>,<,>=,<=,contains,matches}
#               不暴露 lambda / 不 eval / 不 exec;非法表达式 → NodeOutput.error
# - loop.py   → iterate config["iterations"] 次,body 由 executor 决定
```

### 3.4 FastAPI 路由

| Method | Path | 入参 | 出参 | 权限 |
|---|---|---|---|---|
| GET | `/api/workflows?workspace_id=...` | - | `{workflows: [...]}` | `require_user` |
| POST | `/api/workflows` | `{name, workspace_id, nodes, edges}` | `Workflow` | `require_owner_user` |
| GET | `/api/workflows/{id}` | - | `Workflow` | `require_user` + workspace_member |
| PUT | `/api/workflows/{id}` | `{name?, nodes?, edges?, status?}` | `Workflow`(version 自增) | workspace_member |
| DELETE | `/api/workflows/{id}` | - | `{success: true}` | workspace_member |
| POST | `/api/workflows/{id}/execute` | `{inputs, workspace_id}` | `{outputs, steps, total_tokens, total_duration_ms}` | workspace_member |
| GET | `/api/workflows/{id}/versions` | - | `{versions: [...]}` | workspace_member |
| POST | `/api/workflows/{id}/rollback/{version}` | - | `Workflow`(新 version) | workspace_member |

**响应统一信封**(继承 v1.5.10 `patterns.md`):
- 成功:对象或 `{items: [...]}` 数组
- 失败:`{error: {code: "...", message: "...", details: {...}}}`

## §4 错误处理

| 层 | 错误类型 | 处理 |
|---|---|---|
| API 入参校验 | Pydantic 422 | 顶层 `validation_exception_handler` 统一信封 `{error: {code: "INVALID_INPUT", details}}` |
| 权限 | `require_user` 401 / `require_workspace_member` 403 | 同上 envelope |
| 工作流节点失败 | NodeOutput.error 非 None | `fail_fast=False`(默认)→ 记 step.error,workflow 继续;`fail_fast=True` → 中止,返回 `{outputs: null, failed_node_id, steps}` |
| LangGraph agent 失败 | lead_agent 抛异常 | NodeOutput.error + step.status="failed" |
| 配额超额 | QuotaDecision.allowed=False | 429 `{error: {code: "QUOTA_EXCEEDED", mode: "hard"}}` |
| 跨部门发布权限 | 用户不在 target_workspace | 403 `{error: {code: "NOT_WORKSPACE_MEMBER"}}` |
| BRANCH DSL 非法 | AST 解析失败 | NodeOutput.error + step.status="failed";**不**让进程 crash |
| LOOP 死循环 | iterations > 1000 / 单次 wall-clock > 60s | `LoopNode.__post_init__` 拒入;executor 设 60s 超时 → NodeOutput.error |
| SQLite 写失败 | sqlite3.OperationalError | 500 + 日志;**不**自动降级 InMemory(避免状态分裂) |
| Nginx 限流 | 503 | 前端显示通用 `connectionFailed`(v1.5.10 已有) |

| 5 | 5+ 连接器全部可用(飞书/钉钉/企微/邮件 + Jira/Linear/Slack) | 连接器集成测试全绿 + 灰度部门实际使用 1 周 | C 线 |
| 6 | P95 < 2s @ 50 并发 | k6 脚本:50 VU 持续 5 min,POST /execute 线性 workflow | `performance-optimizer` |

## §7 风险与缓解(继承战略 spec §8 + 增补)

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 大版本延期(5-7 周 → 8+ 周) | 中 | 高 | §1 P0/P1/P2 切分;每周 burn-down check;P1 Slack/A6 推 v1.6.1 兜底 |
| AGENT 节点调 LangGraph 增加延迟 | 中 | 中 | embed 走 `deerflow.client`(v1.5.9 已稳定),不走 HTTP;P95<2s 验收把门 |
| BRANCH DSL 安全(用户写任意 lambda) | 低 | 高 | 不暴露 lambda;固定 AST 形式 `var op value`,op ∈ {==,!=,>,<,>=,<=,contains,matches};用 AST evaluator,**绝不** eval/exec |
| Workflow 拓扑死循环(LOOP N=∞) | 低 | 中 | `LoopNode.__post_init__` 校验 `iterations ∈ [1, 1000]`;executor 设 wall-clock 超时 60s |
| SQLite 写并发冲突 | 低 | 低 | `check_same_thread=False` + 单写者锁;文档说"灰度兜底,生产换 Postgres" |
| PublishService 链式引用无限增长 | 极低 | 极低 | `published_from_thread_id` 是 id 字符串,空间 O(1);`publish_history` maxlen=50 |
| 5+ 连接器统一协议被 Slack 破坏 | 低 | 中 | Slack 走 v1.5.5 已有 Connector 协议;若不兼容记 P2 推迟 |
| 验收 #4 非工程师测出 UX 灾难 | 中 | 高 | B1 NodeInspector 早期就给产品同事 demo,迭代;**不要"做完再给"** |
| `agents.py` 中 `@dataclass(frozen=True)` 兼容性 | 低 | 低 | Python 3.12 原生支持,无需 `attrs`;CI 跑 ruff + mypy |

## §12 设计决策记录(关键 ADR)

| 决策 | 选择 | 替代方案 | 理由 |
|---|---|---|---|
| AGENT 节点执行路径 | `deerflow.client.DeerFlowClient` embed | 通过 LangGraph HTTP server | 降低延迟,P95 < 2s 验收要求;embed runtime v1.5.9 已稳定 |
| BRANCH 表达式 | 自实现 AST evaluator(`var op value`) | 引入表达式引擎(如 simpleeval) | 不引入新依赖;固定 op 集合 + 严格校验足以覆盖 90% 用例 |
| 默认 fail_fast | False | True | 非工程师调试时,看到 partial outputs 比"全部失败"更有价值;配 `MICX_WORKFLOW_FAIL_FAST=true` 给严格场景 |
| publish_history 上限 | 50 | 无上限 | 防无限增长;50 条足够追溯最近活动;归档到 audit log 可保留更久 |
| ABAC 整条推到 v1.7+ | 不在 v1.6.x 做 | v1.6.x 加简化版 | 战略 spec §4.3 标 P2;v1.5.4 RBAC v2 暂时够用;v1.6.x 真做是浪费 |
| SQLite 持久化标 P1 | 默认 InMemory | 默认 SQLite | InMemory 灰度期足够;SQLite 是"重启不怕"兜底,真生产换 Postgres(战略 spec §5 已规划) |
| Langfuse 真实 SDK 标 P2 | 不在 v1.6.x 做 | v1.6.x 合真 SDK | v1.5.10 留 surface 已经满足灰度观测;真 SDK 等 v1.6.x 灰度跑通再合,避免污染 v1.6.x 验收 |
