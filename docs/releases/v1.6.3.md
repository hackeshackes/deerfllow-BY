## Critical fix: drift recovery helper

`scripts/reset_owner_password.py` covers the scenario the `admin_secrets` cluster
cannot — the owner account whose password hash drifts away from
`BY_ADMIN_PASSWORD` (forgotten, manual hash edit, schema rename). Run with:

```bash
BY_ADMIN_PASSWORD='<plaintext>' python3 scripts/reset_owner_password.py
```

The script regenerates `salt` + PBKDF2-SHA256 (120k iter) matching
`auth.py:_hash_password`, atomically rewrites only the owner record via
`tempfile.mkstemp` + `Path.replace`, leaves every other user (member, invited)
untouched, and emits progress on stdout.

> Verified end-to-end this session: re-seeded owner, `/api/session/login`
> returned `200`, all four `/api/admin/secrets/*` endpoints responded as
> expected.

## admin-secrets M1 — endpoints

Three owner-only FastAPI endpoints backed by the existing Fernet-encrypted
admin vault (cipher key derived from `MICX_ADMIN_SECRET_KEY` /
`BETTER_AUTH_SECRET`).

### `POST /api/admin/secrets/upsert`

Create or replace a vault entry. `value=null` deletes the entry.

```json
{ "key": "models/dspark-v1.1-mida-brikie/api_key", "value": "sk-…" }
```

Returns:

```json
{
  "key": "models/dspark-v1.1-mida-brikie/api_key",
  "reference": "secret://models/dspark-v1.1-mida-brikie/api_key",
  "status": "upserted",
  "masked": true
}
```

### `POST /api/admin/secrets/rotate`

Atomic rotate of the env cipher key **plus** re-encrypted vault. Gated by
`current_admin_password` so a hijacked session cannot lock the owner out.
Sliding-window rate limit: 10 req / minute / IP. 503 if the OS lacks the
lock primitive (POSIX `fcntl.flock` / Windows `msvcrt`).

### `GET /api/admin/secrets/status`

Reports `missing` / `placeholder` / `fresh` / `configured` per catalog key.
`?include_all=true` adds env-only keys (no vault entry needed).

Audit emits `admin_secret.upserted`, `admin_secret.deleted`, `admin_secret.rotated`.

## admin-secrets M2 — cross-process + UI

* `_acquire_rotate_lock()` context manager wraps rotate path with POSIX
  `fcntl.flock` (Windows `msvcrt` fallback), preventing two replicas racing
  on cipher swap.
* `SECRETS_VAULT_ROUTABLE` allow-list restricts `/secrets/rotate` to
  privileged + production vault keys; all other keys return `400` with a
  hint to edit `.env` instead.
* `GET /api/admin/secrets/audit-events` (owner-only) reads `audit.jsonl`
  tail + in-memory filter; plaintext never leaves the backend.
* Frontend `secrets-admin-page.tsx` adds four tabs (status / upsert / rotate
  / audit) behind the existing `admin-page-shell` KeyRoundIcon entry; i18n
  namespace `admin.secrets` translated for `en-US` + `zh-CN`.

## Vendor-aware model catalog

`backend/packages/harness/deerflow/models/` now distinguishes vendor-specific
request shape and override semantics:

* **Anthropic** — `anthropic-beta` header tuple on `claude-*` `3.7`+ / `4.x` models
* **Google** — preserves `thought_signature` / `thoughts` fields on `gemini-2.x` thinking models
* **DeepSeek** — surfaces `reasoning_content` as separate stream chunk for `deepseek-r1` family
* **Qwen** — exposes `thinking_budget` parameter for Qwen3 reasoning models

Tests: `test_model_vendor_catalog.py` — 34 cases covering canonical payloads,
missing-field branches, type mismatches, nested JSON. The catalog references
`models.override.yaml` entries that use `secret://…` URIs, so secret rotation
takes effect on the next request without restart.

## Env template hardening

`.env` / `docker/.env` templates now include `BETTER_AUTH_SECRET` and
`MICX_ADMIN_SECRET_KEY` placeholders with COPY guidance so new deployments
hit the V01 fail-fast path instead of booting with dev defaults. This
complements the runtime vault: the vault handles *runtime* secret rotation;
templates handle *deployment* secret initialization.

---

## Provisioner profile

`docker/docker-compose.yaml` `provisioner` service now carries
`profiles: ["provisioner"]`, matching `docker-compose-dev.yaml`. Production
K8s users run:

```bash
docker compose -f docker/docker-compose.yaml --profile provisioner up -d --build
```

Local sandbox users (the default) no longer build or start `provisioner`
image, which trims image-build time during development.


> (or fallback `BETTER_AUTH_SECRET`) to be set before gateway startup —
> missing or dev-default values cause fail-fast. Make sure `.env` carries a
> stable value before pulling v1.6.3.
