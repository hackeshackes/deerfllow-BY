## What is this project

DeerFlow is a **LangGraph-based AI super-agent** with a full-stack architecture. The backend (port 8001 gateway + port 2024 LangGraph runtime) provides a single-threaded super-agent with sandbox execution, persistent memory, subagent delegation, and an extensible tool/skills/MCP surface. The frontend (port 3000, Next.js 16) is a stateful chat application: users create threads, send messages, and receive streamed AI responses with artifacts and todos. Nginx (port 2026) is the unified entry point.

The v1.6.0-canvas line adds the **visual workflow canvas** — domain experts compose prompt/agent/tool/branch/loop nodes in a React Flow editor, save them as versioned workflows, and run them against the LangGraph runtime with workspace-scoped quota enforcement. A **cross-workspace publish** flow lets a department clone a thread into another department's space, preserving the original lineage. The same line that v1.5.x laid data-plane foundations (multi-tenancy, quotas, observability) for now gets a first-class authoring surface.

## How to deploy

### Local (this repo's `docker/`)

```bash
cd /Users/baoyu/Documents/GitHub/deerfllow-BY
docker compose -f docker/docker-compose.yaml up -d --build
# Frontend image is rebuilt from frontend/Dockerfile. Gateway mounts
# backend/ as a dev volume, so the v1.6.0-canvas executor changes
# are picked up on `docker compose restart gateway` (or auto-reload
# in gateway-mode). LangGraph image is the langgraph dockerfile.
```

### Production / staging

- The `v1.6.0-canvas` tag is the freeze point; pin frontend and gateway deploys to that SHA.
- Roll forward to `v1.6.0-canvafix` once the executor + 23-fail fixes are merged (see CHANGELOG.md).
- Backend `WorkflowStore` is in-memory — restarting the gateway drops unpublished drafts. The `persistence/sqlite_store.py` shim is staged in the v1.6.x plan but not landed; use a sticky session/blue-green deploy until it ships.
- Nginx config in `docker/nginx/nginx.conf` already routes `/api/workflows*` to gateway and `/workspace/workflows*` to frontend. No nginx changes required.
