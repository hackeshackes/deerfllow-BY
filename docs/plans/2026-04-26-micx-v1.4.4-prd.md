# MicX v1.4.4 PRD - 用户端体验升级

**版本目标**: *解决高频痛点，让日常使用更顺滑*

**日期**: 2026-04-26

---

## 一、背景

当前 MicX 功能完整，但日常使用中有几个高频痛点：

1. **新建对话不知道问什么** - 空白输入框，用户不知道从哪开始
2. **输入框太小** - 复杂任务时输入不便，无法上传文件/粘贴长文本
3. **侧边栏找对话费劲** - 对话多了之后难搜索，无法快速找到进行中的对话
4. **上下文切换麻烦** - 想换个模式/模型需要点好几下
5. **知识库和对话割裂** - 想把对话内容存入知识库操作繁琐

---

## 二、对话管理增强

### 2.1 对话置顶

将重要对话固定在列表顶部，方便快速访问。

**API 端点**:
```bash
POST   /api/threads/{id}/pin           # 置顶对话
DELETE /api/threads/{id}/pin           # 取消置顶
GET    /api/threads?pinned=true       # 仅获取置顶对话
```

**数据模型**:
```python
# threads 表新增字段
is_pinned: bool = False
pinned_at: datetime | None
```

**前端行为**:
- 置顶对话显示在列表最顶部，带📌图标
- 支持多对话置顶（按置顶时间倒序）
- 置顶状态跨设备同步

### 2.2 对话归档

将已完成或暂不关注的对话归档，归档后默认不显示。

**API 端点**:
```bash
POST   /api/threads/{id}/archive       # 归档对话
DELETE /api/threads/{id}/archive       # 取消归档
GET    /api/threads?archived=true     # 获取已归档对话
GET    /api/threads?archived=false    # 获取未归档对话（默认）
```

**数据模型**:
```python
# threads 表新增字段
is_archived: bool = False
archived_at: datetime | None
```

**前端行为**:
- 对话列表默认隐藏已归档对话
- 提供"显示已归档"开关
- 归档对话支持取消归档（恢复）
- 已归档对话不计入未读数

### 2.3 对话分组（文件夹）

支持创建文件夹对对话进行分类管理。

**API 端点**:
```bash
GET    /api/thread-folders            # 列出文件夹
POST   /api/thread-folders            # 创建文件夹
PUT    /api/thread-folders/{id}       # 重命名文件夹
DELETE /api/thread-folders/{id}       # 删除文件夹
POST   /api/threads/{id}/folder       # 将对话移入文件夹
DELETE /api/threads/{id}/folder       # 将对话移出文件夹
```

**数据模型**:
```sql
CREATE TABLE thread_folders (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    user_id TEXT NOT NULL,
    workspace_id TEXT,
    color TEXT DEFAULT '#6366f1',
    sort_order INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**前端行为**:
- 左侧边栏显示文件夹列表（收起/展开）
- 支持拖拽对话到文件夹
- 支持拖拽调整文件夹顺序
- 支持文件夹颜色自定义
- 无文件夹的对话显示在"全部对话"下

## 六、数据库迁移

```sql
-- 对话置顶/归档
ALTER TABLE threads ADD COLUMN is_pinned BOOLEAN DEFAULT FALSE;
ALTER TABLE threads ADD COLUMN pinned_at TIMESTAMP;
ALTER TABLE threads ADD COLUMN is_archived BOOLEAN DEFAULT FALSE;
ALTER TABLE threads ADD COLUMN archived_at TIMESTAMP;

-- 文件夹
CREATE TABLE IF NOT EXISTS thread_folders (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    user_id TEXT NOT NULL,
    workspace_id TEXT,
    color TEXT DEFAULT '#6366f1',
    sort_order INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

ALTER TABLE threads ADD COLUMN folder_id TEXT REFERENCES thread_folders(id);

CREATE INDEX IF NOT EXISTS idx_threads_pinned ON threads(is_pinned);
CREATE INDEX IF NOT EXISTS idx_threads_archived ON threads(is_archived);
CREATE INDEX IF NOT EXISTS idx_thread_folders_user ON thread_folders(user_id);
```
