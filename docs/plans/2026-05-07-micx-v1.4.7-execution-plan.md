## 二、技术方案详解

### 2.1 后端结构

```
backend/app/gateway/routers/ppt.py          # API 端点
backend/app/gateway/routers/tasks.py        # 复用任务系统
backend/services/ppt/
  ├── __init__.py
  ├── generator.py                         # PPT 生成器
  ├── templates.py                         # 模板管理
  └── cleanup.py                           # 过期文件清理
```

### 2.2 ppt-master 集成

ppt-master 安装路径：
```
backend/third_party/ppt-master/
  ├── skills/ppt-master/                   # skill 文件
  ├── scripts/                             # Python 脚本
  └── templates/                           # 模板文件
```

调用方式（Python）：
```python
import subprocess
result = subprocess.run([
    "python3", "skills/ppt-master/scripts/project_manager.py",
    "init", project_name, "--format", "ppt169"
], cwd=ppt_master_dir, capture_output=True)
```

### 2.3 API 设计

#### POST /api/ppt/generate
Request:
```json
{
  "source_type": "conversation_summary | knowledge_doc | text",
  "content": "PPT 内容（Markdown 或纯文本）",
  "style": "auto | standard | academic | consulting | tech",
  "options": {
    "page_count": "auto | 8 | 12 | 15",
    "animation": true,
    "notes": false
  }
}
```

Response:
```json
{
  "task_id": "ppt_xxx",
  "status": "pending",
  "estimated_time": 60
}
```

#### GET /api/ppt/status/{task_id}
Response:
```json
{
  "task_id": "ppt_xxx",
  "status": "generating | completed | failed",
  "progress": 50,
  "output": {
    "file_id": "file_xxx",
    "filename": "presentation_20260101.pptx",
    "size": 1024000,
    "download_url": "/api/ppt/download/file_xxx"
  },
  "error": null
}
```

### 2.4 前端结构

```
frontend/src/components/workspace/ppt/
  ├── ppt-generator-dialog.tsx    # 主弹窗
  ├── ppt-preview-card.tsx       # 预览卡片
  ├── style-selector.tsx         # 风格选择器
  └── index.ts

frontend/src/core/ppt/
  ├── api.ts                     # API 调用
  ├── hooks.ts                   # React hooks
  └── types.ts                   # 类型定义
```

## 五、注意事项

### 5.1 ppt-master 依赖

ppt-master 需要 Python 3.10+ 环境，主要依赖：

```txt
python-docx>=1.0.0
python-pptx>=3.3.0
pillow>=10.0.0
requests>=2.31.0
beautifulsoup4>=4.12.0
lxml>=4.9.0
svgwrite>=1.5.0
```

可能需要 playwright（可选）：
```bash
playwright install chromium
```

### 5.2 文件大小限制

- 输入内容限制：100KB（纯文本），避免 token 溢出
- 输出 PPT 限制：50MB
- 存储清理：7 天后自动删除

### 5.3 并发限制

- 每用户同时生成任务：最多 2 个
- 全局并发任务：最多 10 个
- 队列满时返回 429 Too Many Requests

### 5.4 沙箱 vs 后端服务

**推荐方案：后端服务模式（方案 A）**

原因：
1. 沙箱执行 Python 脚本复杂度高
2. 后端服务更稳定可靠
3. 可复用现有任务队列
4. 便于监控和限流

ppt-master 将在后端环境中安装，作为本地服务调用。
