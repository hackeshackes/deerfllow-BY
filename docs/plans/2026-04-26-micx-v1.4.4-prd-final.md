# MicX v1.4.4 PRD - 空间化用户工作台

**版本**: v1.4.4  
**主题**: 空间化用户工作台  
**目标**: 让普通用户清楚知道自己在哪个空间、AI 会参考什么资料、结果会沉淀到哪里、哪些内容会被共享。  
**日期**: 2026-04-26  
**状态**: 待确认

---

## 一、版本判断

MicX 已经具备多用户、工作区、对话、知识库、定时任务、技能、IM 渠道、Admin 管理等基础能力。v1.4.4 不应继续堆功能，而应解决用户端的组织方式问题。

当前用户端像一组功能菜单：对话、知识库、定时任务、技能、设置。用户需要自己判断：

- 当前是在个人空间还是共享空间？
- 这个对话谁能看到？
- AI 这次会参考哪些资料？
- 定时任务执行后结果去哪？
- 技能是个人偏好，还是团队共享工作流？

v1.4.4 的核心是建立统一心智：

> 个人空间用于私人思考和实验。  
> 共享空间用于团队协作和沉淀。  
> 对话、资料、自动化、工作流都跟随当前空间。  
> 私人内容不会默认泄漏到共享空间，发布必须显式发生。

---

## 二、目标用户

### 2.1 普通成员

典型行为：

- 跟 AI 做私人对话
- 上传文件，让 AI 帮忙分析
- 使用团队共享的知识库和工作流
- 查看团队共享对话和自动化结果

痛点：

- 不知道内容是否会被别人看到
- 不知道哪个知识库会被 AI 使用
- 技能太像开发者配置，不像可直接使用的能力
- 定时任务概念太技术化

### 2.2 团队负责人 / 空间管理员

典型行为：

- 管理共享空间资料
- 创建团队工作流
- 设置周期性自动化，如日报、周报、项目风险扫描
- 让外部渠道对话进入团队空间

痛点：

- 团队知识、自动化、对话沉淀分散
- 无法快速看到共享空间最近发生了什么
- 很难推动团队成员复用同一套工作流

### 2.3 系统管理员

v1.4.4 不重点增强系统管理员。Admin 能力已在 v1.4.3 加强，本版本只确保用户端体验和 Admin 权限模型不冲突。

## 三、范围

### 3.1 本版本必须交付

1. **一级空间切换**：个人空间 / 共享空间成为左侧栏顶部的一级上下文。
2. **空间化导航**：对话、资料库、自动化、工作流都跟随当前空间。
3. **对话分区**：我的对话、共享对话、自动化结果、外部渠道对话分开显示。
4. **新建对话场景化**：新建时显示当前空间、资料来源、工作流入口。
5. **知识库用户侧重命名为资料库 / 资料来源**：弱化数据库管理心智。
6. **定时任务用户侧重命名为自动化**：保留 `/workspace/tasks` 兼容，新增 `/workspace/automations` 用户入口。
7. **技能用户侧重组为工作流**：普通用户用卡片启动，不直接面对 SKILL.md。

### 3.2 本版本可选交付

1. AI 回复一键存入资料库。
2. 从对话创建自动化。
3. 工作流绑定默认资料库和模式。

### 3.3 本版本不做

1. 复杂文件夹和拖拽整理。
2. 复杂全文搜索和向量检索重构。
3. 实时多人共同输入。
4. 技能市场。
5. 钉钉 / 企业微信新工具建设。

这些放到 v1.4.5 之后。

## 四、信息架构

### 4.1 左侧栏结构

当前左侧栏：

```text
WorkspaceHeader
Chats / Agents
Recent Chats
Scheduled Tasks
Settings and More
```

v1.4.4 改为：

```text
MicX
[当前空间 ▼]

+ 新建对话
对话
资料库
自动化
工作流

当前空间对话
├── 最近继续
├── 自动化结果
└── 外部渠道

底部：账号 / 设置 / Admin / 退出
```

### 4.2 空间切换行为

空间切换不再隐藏在设置菜单，而是左侧栏顶部一级按钮。

空间类型：

| 类型 | 展示名 | 默认内容可见性 |
|------|--------|----------------|
| personal | 个人空间 | 仅自己可见 |
| workspace | 共享空间 | 空间成员可见 |

切换后影响：

- 对话列表刷新
- 资料库列表刷新
- 自动化列表刷新
- 工作流推荐刷新
- 新建对话默认归属刷新

## 五、功能需求

### 5.1 一级空间切换

#### 用户故事

作为用户，我希望一眼看到当前是在个人空间还是共享空间，避免把私人内容误发到团队空间。

#### 需求

1. 左侧栏顶部显示当前空间名称。
2. 点击后展示所有可访问空间。
3. 空间项明确显示：个人空间 / 共享空间。
4. 切换空间后刷新当前页面和数据。
5. 如果当前页面不适用于新空间，跳转到 `/workspace/chats/new`。

#### 验收

- 用户不进入设置菜单也能切换空间。
- 当前空间在所有 workspace 页面可见。
- 切换后 Recent Chat List 不混入其他空间内容。

---

### 5.2 对话分区

#### 用户故事

作为用户，我希望私人对话、团队对话、自动化结果、外部渠道对话分开，避免最近列表变成垃圾堆。

#### 分区规则

| 分区 | 条件 |
|------|------|
| 最近继续 | source = manual 且当前空间可见 |
| 自动化结果 | source = automation 或 task_id 存在 |
| 外部渠道 | source = channel 或 channel metadata 存在 |
| 已发布 | personal 空间内 visibility = workspace 的对话 |

#### Metadata 约定

新增或规范化：

```json
{
  "space_type": "personal | workspace",
  "source": "manual | automation | channel",
  "task_id": "optional",
  "channel": "feishu | slack | telegram | optional",
  "published_from_thread_id": "optional",
  "last_actor": "user | agent | automation | channel"
}
```

#### 验收

- 个人空间默认只显示用户自己的对话。
- 共享空间默认显示当前 workspace 可见对话。
- 自动化结果在单独分区显示。
- 外部渠道对话在单独分区显示。
- 对话卡片明确显示“私有 / 共享 / 自动化 / 渠道”。

---

### 5.3 新建对话场景化

#### 用户故事

作为用户，我希望新建对话时能选择“这次要参考什么资料”和“用哪个工作流”，而不是面对空白输入框。

#### 页面结构

`/workspace/chats/new`：

```text
当前空间：产品团队空间

你想做什么？
[自由对话] [基于资料问答] [写文档] [分析文件] [创建自动化]

资料来源
[ ] 产品文档库  [ ] 客户反馈库  [ ] 公司制度库

常用工作流
[会议纪要] [周报总结] [需求评审] [代码审查]

输入框
```

#### 行为

1. 个人空间默认不勾选共享资料。
2. 共享空间默认推荐当前空间资料库。
3. 用户选择资料库后，将 knowledge base ids 写入 thread context 或 metadata。
4. 用户选择工作流后，自动填入 prompt starter 和 mode。
5. “创建自动化”跳转到自动化创建页，并带上当前空间。

#### 验收

- 新建页清楚显示当前空间。
- 可选择资料来源。
- 可选择工作流模板启动对话。
- 不选择任何东西时仍可自由对话。

---

### 5.4 资料库：用户侧知识库重组

#### 用户故事

作为用户，我希望知道 AI 当前可使用哪些资料，以及这些资料是个人的还是团队的。

#### 命名

- 用户侧：资料库 / 资料来源
- Admin 侧：知识库管理
- API 和内部代码可继续使用 knowledge 命名

#### 用户侧页面

`/workspace/knowledge` 保留路由，页面标题改为“资料库”。

Tabs：

1. 当前空间资料
2. 我的私人资料
3. 全局资料

每个卡片展示：

- 名称
- 可见范围
- 文档数
- 最近更新
- 是否当前空间默认使用

#### 新增动作

1. 上传资料
2. 设为当前空间默认资料
3. 复制到个人空间
4. 发布到共享空间

#### 验收

- 用户能看懂每个资料库的可见范围。
- 共享空间只推荐当前空间资料。
- 全局资料只读展示。
- 对话创建页能选择这些资料。

---

### 5.5 自动化：用户侧定时任务重组

#### 用户故事

作为用户，我想设置“每天/每周自动让 AI 做某件事”，并清楚看到结果发到哪里。

#### 命名

- 用户侧：自动化
- 技术/API：tasks 可保留

#### 路由

- 新增 `/workspace/automations`
- 新增 `/workspace/automations/new`
- 旧 `/workspace/tasks` 保留跳转或兼容

#### 列表字段

自动化卡片展示：

- 名称
- 所属空间
- 触发频率，例如“每天 09:00”
- 资料来源
- 输出位置：个人对话 / 共享空间 / webhook
- 上次运行状态
- 下次运行时间

#### 创建页用户语言

替代直接暴露 Cron：

```text
运行频率：
[每天] [每周] [每月] [自定义]

运行时间：09:00

输出到：
[新建个人对话] [当前共享空间] [指定对话]
```

高级模式保留 Cron 输入。

#### 验收

- 普通用户不需要理解 Cron 也能创建自动化。
- 自动化明确显示输出位置。
- 自动化结果对话进入“自动化结果”分区。
- 旧任务仍能正常显示和运行。

---

### 5.6 工作流：用户侧技能重组

#### 用户故事

作为用户，我希望看到的是“会议纪要、周报总结、代码审查”这样的能力，而不是技能配置文件。

#### 页面

新增或重命名 `/workspace/settings/skills` 为用户入口“工作流”。

Tabs：

1. 常用
2. 当前空间推荐
3. 团队共享
4. 系统内置
5. 我的创建

#### 卡片字段

- 工作流名称
- 它能帮我做什么
- 推荐场景
- 是否启用
- 使用按钮
- 设为常用
- 创建自动化

#### 行为

1. 点击“使用”进入新建对话并带上 prompt starter。
2. 点击“创建自动化”进入自动化创建页并带上该工作流。
3. 高级用户仍可进入 SKILL.md 编辑。

#### 验收

- 普通用户可从工作流卡片直接开始对话。
- 工作流能在新建对话页显示。
- 工作流启用/禁用仍复用现有 user skills API。

## 六、数据与 API 需求

### 6.1 Threads

现有：

- `metadata.visibility`
- `metadata.owner_user_id`
- `metadata.workspace_id`
- `GET/POST/PATCH /api/threads/search`

新增查询能力：

```bash
POST /api/threads/search
{
  "metadata": {
    "source": "automation"
  },
  "limit": 100,
  "offset": 0
}
```

若已有 metadata exact match 足够，优先复用；不新增 API。

新增 metadata 由创建/同步时写入。

### 6.2 Knowledge

现有：

- `workspace_id`
- `visibility`
- `is_global`
- `knowledge_shares`

新增可选字段：

```sql
ALTER TABLE knowledge_bases ADD COLUMN is_default_for_workspace BOOLEAN DEFAULT FALSE;
```

如实现复杂，可 v1.4.4 先不落库，用前端推荐当前空间资料即可。

### 6.3 Tasks / Automations

现有：

- `workspace_id`
- `visibility`
- `thread_id`
- `output_config`

建议扩展 `output_config`：

```json
{
  "save_to_thread": true,
  "target": "personal_thread | workspace_thread | existing_thread | webhook",
  "target_thread_id": "optional",
  "target_workspace_id": "optional"
}
```

### 6.4 Skills / Workflows

现有：

- `/api/user/skills`
- enable/disable
- share/unshare
- custom skill creation

v1.4.4 不新增技能后端模型。前端通过现有字段包装为 workflow cards。

## 七、权限规则

### 7.1 对话

- 个人空间创建：默认 private。
- 共享空间创建：默认 workspace。
- private 对话仅 owner 可见。
- workspace 对话当前 workspace 成员可见。
- 发布到共享空间必须显式点击。

### 7.2 资料库

- private：仅 owner 可见。
- workspace：当前 workspace 可见。
- global：所有用户可见，普通用户只读。

### 7.3 自动化

- private 自动化仅 owner 可见和执行。
- workspace 自动化当前 workspace 可见。
- 执行结果可写入个人或共享空间，必须在创建时明确。

### 7.4 工作流

- 用户可启用/禁用可见工作流。
- 用户创建的工作流默认 private。
- 共享工作流进入当前 workspace 推荐区。

## 八、文案原则

用用户语言替代技术语言。

| 旧文案 | 新文案 |
|--------|--------|
| Knowledge Base | 资料库 / 资料来源 |
| Scheduled Task | 自动化 |
| Skill | 工作流 |
| Cron | 自定义时间规则 |
| visibility | 可见范围 |
| workspace | 共享空间 |

Admin 页面仍可使用技术文案。

## 九、验收标准

v1.4.4 完成后，用户应能一眼回答：

1. 我现在在哪个空间？
2. 这个对话谁能看到？
3. AI 这次会参考哪些资料？
4. 自动化结果会发到哪里？
5. 这个工作流是我个人的，还是团队共享的？

### 功能验收清单

- [ ] 左侧栏顶部显示当前空间并可切换。
- [ ] 对话列表按当前空间过滤。
- [ ] 对话列表按来源分区：最近继续 / 自动化结果 / 外部渠道。
- [ ] 新建对话页显示当前空间。
- [ ] 新建对话页可选择资料来源。
- [ ] 新建对话页可选择工作流。
- [ ] 知识库用户侧显示为资料库。
- [ ] 资料库卡片展示可见范围。
- [ ] 定时任务用户侧显示为自动化。
- [ ] 自动化创建页支持非 Cron 表单。
- [ ] 技能用户侧显示为工作流卡片。
- [ ] 所有旧路由兼容，不破坏现有链接。

## 十、成功指标

定性指标：

- 用户不再问“这个在哪里设置”。
- 用户不再误把私人对话共享。
- 用户知道资料库和对话的关系。
- 用户愿意从工作流入口开始任务。

可量化指标（后续埋点）：

- 新建对话页工作流点击率
- 资料来源选择率
- 自动化创建成功率
- 共享空间对话访问率
- 工作流使用次数
