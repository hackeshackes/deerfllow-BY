# MicX v1.4.7 PRD - 智能演示生成

**版本**: v1.4.7
**主题**: 智能演示生成
**目标**: 让用户基于对话内容、知识库文档或任意文本，一键生成专业级可编辑 PPTX。
**日期**: 2026-05-07
**状态**: 草案

---

## 一、版本判断

v1.4.6 完成了"可信知识与记忆"——用户能看到 AI 用了什么资料、记住了什么信息。

v1.4.7 要回答的问题是：

> **用户能不能把 AI 分析结果、对话摘要、知识库内容直接变成专业 PPT？**

现在用户的痛点是：

1. **手动复制粘贴** — AI 分析完一个报告后，用户要手动把结论复制到 PPT 里，效率低且容易出错。

2. **格式不统一** — 对话摘要直接贴到 PPT 里没有设计感，需要花时间排版。

3. **PPT 工具割裂** — 用户需要在 MicX 和专业 PPT 工具之间来回切换。

v1.4.7 的核心判断：

> **一个平台完成从对话分析到专业演示的全流程。**

本版本的主轴是 **AI 分析 → 一键生成专业 PPT**：

1. 一键生成：基于对话摘要、知识库文档或任意文本生成 PPT。
2. 专业设计：内置多种模板和设计风格，无需用户手动排版。
3. 真正可编辑：输出真正的 PPTX 格式（不是图片拼贴），每个元素都可在 PowerPoint 中编辑。
4. 保持上下文：PPT 生成与 MicX 工作流无缝衔接。

## 二、目标用户

### 2.1 知识工作者

典型行为：

- 用 AI 分析了一份行业报告，希望把关键结论分享给团队。
- 完成了项目复盘，需要向领导汇报。
- 整理了客户需求文档，需要做成演示文稿。

痛点：

- 手动从 AI 输出复制内容到 PPT 太费时间。
- AI 的输出直接贴 PPT 格式很丑。
- 需要在多个工具之间切换。

### 2.2 团队负责人

典型行为：

- 查看团队的 AI 分析摘要，需要转成团队会议 PPT。
- 想用团队模板统一 PPT 风格。

痛点：

- 团队成员制作的 PPT 风格不统一。
- 模板复用困难。

### 2.3 培训/教育场景

典型行为：

- 用 AI 整理了培训资料，需要做成学员版 PPT。
- 教学演示需要图文并茂的 PPT。

痛点：

- 从长文档整理 PPT 工作量大。
- 需要插入配图增加可读性。

## 四、核心功能

### 4.1 PPT 生成入口

#### 4.1.1 对话摘要页入口

在 `thread-summary-dialog.tsx`（v1.4.5 已实现）增加"生成 PPT"按钮：

- 用户完成对话后，点击摘要气泡中的"生成 PPT"按钮。
- 系统将摘要内容发送给 ppt-master 生成 PPT。
- 生成完成后提供下载链接。

**UI 位置**：摘要气泡下方操作栏。

#### 4.1.2 知识库文档入口

在知识库文档列表页和详情页增加"生成 PPT"按钮：

- 针对单个文档或整个知识库内容生成 PPT。
- 支持选择模板风格。

**UI 位置**：文档卡片操作栏、知识库详情页顶部。

#### 4.1.3 侧边栏入口

在左侧导航栏增加"智能演示"或"PPT"入口：

- 用户可上传文本/文档直接生成 PPT。
- 独立的 PPT 创建工作区。

**UI 位置**：侧边栏底部或新建按钮下拉菜单。

### 4.2 ppt-master 集成方案

#### 4.2.1 架构选择

ppt-master 是一个本地运行的 skill，工作在 AI IDE 环境中。有三种集成路径：

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **方案 A: 后端服务** | 在 backend 部署 ppt-master 作为 HTTP 服务 | 可被所有客户端调用，稳定 | 需要维护服务，有额外依赖 |
| **方案 B: Agent Skill** | 将 ppt-master 封装为 DeerFlow Skill，Agent 可调用 | 原生集成，自然语言触发 | 需要沙箱支持 Python 脚本执行 |
| **方案 C: 前端 + API** | 前端调用后端 API，后端调用 ppt-master | 最灵活，可显示生成进度 | 复杂度最高 |

**推荐方案：B**（Agent Skill）+ 备选方案 A（后端服务）

#### 4.2.2 集成路径详解

**方案 B 实施步骤**：

1. 将 ppt-master 安装到 `skills/public/` 目录。
2. 配置 DeerFlow 沙箱支持 Python 脚本执行（ppt-master 需要 Python 环境）。
3. 在前端增加 PPT 生成入口（对话摘要、知识库、侧边栏）。
4. 用户触发时，Agent 调用 ppt-master skill 生成 PPT。
5. 生成完成后，PPT 保存到用户工作区，提供下载链接。

**方案 A 备选实施步骤**：

1. 在 backend 新增 `/api/ppt/generate` 端点。
2. 后端调用 ppt-master Python 脚本，传递源内容。
3. 返回生成的 PPTX 文件路径或直接返回文件流。
4. 前端提供下载或在线预览。

#### 4.2.3 依赖管理

ppt-master 依赖：

```
python-docx>=1.0.0
python-pptx>=3.3.0
pillow>=10.0.0
requests>=2.31.0
beautifulsoup4>=4.12.0
playwright>=1.40.0  # 可选，用于网页转 Markdown
pillow-avif-plugin>=1.0.0  # 可选
```

需要在 DeerFlow 沙箱或 backend 环境中安装。

### 4.3 PPT 生成配置

#### 4.3.1 内容来源

| 来源 | 说明 |
|------|------|
| 对话摘要 | 取 thread-summary-dialog 生成的摘要内容 |
| 知识库文档 | 取指定 KB 的文档内容（支持 TXT/Markdown） |
| 用户输入 | 用户直接粘贴或输入的文本内容 |
| 上传文件 | 支持 PDF、DOCX、Markdown 文件 |

#### 4.3.2 模板选择

ppt-master 支持多种模板和风格：

| 风格 | 适用场景 |
|------|----------|
| Free Design | 默认，让 AI 选择最佳设计 |
| Academic | 学术答辩、研究报告 |
| Consulting | 咨询报告、PPT 风格 |
| Tech/SaaS | 科技产品发布 |
| Magazine | 杂志风格、图文混排 |
| Dark Art | 暗黑风格、艺术分析 |
| Nature Documentary | 自然纪录片风格 |

**用户选项**：

- 自动推荐（基于内容类型）
- 手动选择风格
- 自定义模板（未来版本）

#### 4.3.3 生成参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 页数 | 8-15 页 | 根据内容长度自动调整 |
| 格式 | 16:9 | 支持 4:3 |
| 是否加动画 | 是 | 默认启用淡入动画 |
| 是否加备注 | 否 | 可选生成演讲者备注 |
| 是否加配音 | 否 | 未来版本 |

### 4.4 PPT 输出与管理

#### 4.4.1 输出格式

- 主文件：原生可编辑 PPTX（`.pptx`）
- 备份文件：SVG 快照（用于视觉参考）

#### 4.4.2 存储位置

- 用户个人空间：`/workspaces/{user_id}/ppt/`
- 命名格式：`{project_name}_{timestamp}.pptx`

#### 4.4.3 下载与分享

- 提供下载链接（有效期 7 天）。
- 可保存到知识库作为资料。
- 可通过 IM 渠道分享（飞书、Slack 等）。

### 4.5 用户体验流程

#### 4.5.1 标准流程（对话摘要 → PPT）

```
用户完成对话
    ↓
点击"查看摘要"
    ↓
摘要气泡出现，下方有"生成 PPT"按钮
    ↓
点击"生成 PPT"
    ↓
弹出配置对话框：
  - 内容来源：对话摘要（已选）
  - 风格选择：自动推荐 / 手动选择
  - 高级选项：页数、动画、备注
    ↓
点击"生成"
    ↓
显示生成进度（Agent 调用 ppt-master）
    ↓
生成完成，显示：
  - PPT 预览缩略图
  - "下载 PPTX" 按钮
  - "保存到资料库" 按钮
```

#### 4.5.2 知识库 → PPT 流程

```
用户浏览知识库
    ↓
选择文档，点击"生成 PPT"按钮
    ↓
弹出配置对话框（内容已选）
    ↓
点击"生成"
    ↓
后续同标准流程
```

## 九、竞争分析

### 9.1 现有方案

| 工具 | PPT 可编辑 | 集成到工作流 | 中文支持 | 成本 |
|------|------------|--------------|----------|------|
| Gamma.app | ❌ 图片级 | ❌ | 一般 | 付费订阅 |
| Microsoft Copilot | ✅ | ❌ | 一般 | 需 M365 |
| Tome | ❌ 图片级 | ❌ | 差 | 付费订阅 |
| **MicX + ppt-master** | **✅ 原生编辑** | **✅ 无缝** | **✅ 完整** | **AI API 费用** |

### 9.2 MicX 优势

1. **真正的 PPTX** — 不是图片拼接，每个元素可编辑
2. **成本低** — 只需 AI API 费用，无额外订阅
3. **数据本地** — 无需上传到第三方
4. **工作流整合** — 在 MicX 对话/知识库中直接生成

## 十、风险与缓解

### 10.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| ppt-master 沙箱执行问题 | PPT 生成失败 | 方案 A 作为备选（后端服务） |
| Python 依赖冲突 | 环境问题 | 使用独立虚拟环境 |
| 长文本截断 | 内容丢失 | 分段处理，控制 token 数 |

### 10.2 用户体验风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 生成时间过长 | 用户流失 | 显示进度条，预估时间 |
| 模板不匹配 | 用户不满意 | 提供"自动推荐"选项 |
| 下载文件找不到 | 用户困惑 | 增强通知，提供多个下载入口 |

## 十一、非功能需求

### 11.1 性能

- PPT 生成：< 2 分钟（15 页以内）
- API 响应时间：< 3 秒（不含生成时间）
- 同时生成任务数：建议限制 3 个/用户

### 11.2 安全

- PPT 生成内容不包含敏感信息（仅发送给 AI API）
- 下载链接有时效性（7 天）
- 文件存储在用户隔离目录

### 11.3 可用性

- 生成失败时提供明确的错误信息和重试选项
- 支持移动端查看下载（不支持生成）
