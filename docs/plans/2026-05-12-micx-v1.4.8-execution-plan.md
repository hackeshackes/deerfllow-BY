### 2.3 Bug 3: 复制功能不生效 ✅
**文件**: `frontend/src/components/workspace/copy-button.tsx`
**修复**: 添加 async/await + error handling
```typescript
const handleCopy = useCallback(async () => {
  try {
    await navigator.clipboard.writeText(clipboardData);
    setCopied(true);
    setTimeout(() => setCopied(false), 2000);
  } catch (err) {
    console.error("Failed to copy:", err);
  }
}, [clipboardData]);
```

### 2.4 Bug 4: reasoning 模型选中态错误 ✅
**文件**: `frontend/src/components/workspace/input-box.tsx`
**修复**: 添加 `isSelectingModeRef` 防止 auto-selection 覆盖用户选择

**问题**: 多次点击不同的历史对话后，会自动跳转到新对话

**验证步骤**:
1. 打开 MicX，进入对话列表
2. 快速切换 3-5 个不同的历史对话
3. 观察是否会自动跳转到新对话

**根因分析方向**:
- `useThreadChat()` 状态管理
- `workspace/chats/page.tsx` 导航逻辑

**可能修复方案**:

- 方案B: 添加 debounce 防止快速切换
- 方案C: 修复路由参数处理

## 五、Phase 4: Docker 优化 (Day 3)

### 5.1 问题分析

**当前状态**: Docker 镜像 4.5G

**原因分析**:
1. dev target 包含完整编译工具链
2. 多阶段构建未正确使用 production target
3. 可能存在缓存问题

### 5.2 优化方案

**方案 A**: 确保使用 production target
```yaml
# docker-compose-dev.yaml
langgraph:
  build:
    target: runtime  # 使用 runtime 而非 dev
```

**方案 B**: 创建分离的 docker-compose 配置
- 开发: dev target，保留工具链
- 生产: runtime target，精简体积

### 5.3 优化目标

| 指标 | 当前 | 目标 |
|------|------|------|
| 镜像体积 | 4.5G | < 2G |
| 构建时间 | > 30min | < 15min |
