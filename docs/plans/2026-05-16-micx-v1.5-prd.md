# MicX v1.5 PRD - 用户体验与功能增强

**版本日期**: 2026-05-16
**状态**: 待用户确认
**优先级**: P0

---

## 1. 背景与目标

当前 MicX 存在以下问题需要解决：

| 问题 | 现状 | 目标 |
|------|------|------|
| 知识库/定时任务入口缺失 | 用户无法直接访问自己的知识库和任务 | 整合到用户工作空间内，独立管理 |
| 技能管理不完整 | 用户无法编辑/删除自己创建的技能 | 支持用户编辑和删除自定义技能 |
| MCP/IM 配置分散 | MCP 和渠道配置分散在不同页面 | 统一到模型管理下，UI 化配置 |
| IM 渠道配置不友好 | 只能通过 config.yaml 配置 | 支持图形化配置界面 |

**本版本目标**：
1. 将知识库和定时任务整合到用户工作空间，支持工作空间内独立管理和跨空间共享
2. 完善用户技能管理（编辑/删除）
3. Admin MCP 配置界面增强，支持预设和自定义配置
4. IM 渠道图形化配置（飞书/Slack/Telegram/钉钉/企微）

## 2. 功能需求详述

### 2.1 知识库与定时任务用户化

#### 2.1.1 用户入口设计

**位置**: `/workspace/settings` 或 `/workspace/more` 下

**导航结构**:
```
设置 (Settings)
├── 个人信息
├── 技能管理
├── 知识库管理 ← 新增（用户自己的知识库）
├── 定时任务 ← 新增（用户自己的任务）
├── 通知设置
└── ...
```

或放在"更多"入口:
```
更多 (More)
├── 知识库
├── 定时任务
├── 共享知识库
├── 共享任务
└── ...
```

#### 2.1.2 权限体系

| 对象 | 创建者权限 | 其他工作区成员 | 管理员权限 |
|------|-----------|----------------|------------|
| 个人知识库 | 增删改查、上传文档、检索 | 无 | 增删改查所有 |
| 个人定时任务 | 增删改查、暂停恢复、执行 | 无 | 增删改查所有、执行 |
| 共享知识库(workspace) | 增删改查、分享 | 查看、使用 | 增删改查、回收 |
| 共享任务(workspace) | 增删改查、分享 | 查看 | 增删改查、回收 |

**可见性选项**:
- `private` - 私有，仅创建者可见
- `workspace` - 工作区共享，工作区成员可查看使用
- `global` - 全局（仅管理员可创建 global 知识库）

#### 2.1.3 分享机制

用户可以将自己的知识库或任务分享到其他工作空间：
- 分享时指定目标工作空间
- 被分享的工作空间成员获得使用权限（非编辑权限）
- 管理员可以回收任何分享

#### 2.1.4 Admin 管理

管理员可在 Admin 控制台查看所有用户的知识库和任务：
- `/workspace/admin/knowledge` - 知识库管理
- `/workspace/admin/tasks` - 定时任务管理
- 支持按用户筛选
- 支持修改可见性（private/workspace/global）

### 2.3 Admin MCP 配置增强

#### 2.3.1 导航结构调整

**新结构**:
```
模型管理
├── 模型配置
├── 技能管理
├── MCP 服务器 ← 现有

    ├── Telegram 配置
    ├── 钉钉配置
    └── 企业微信配置
```

#### 2.3.2 MCP 服务器图形化配置

**功能**:
1. **服务器列表** - 显示所有已配置的 MCP 服务器
2. **添加服务器** - 支持 STDIO/SSE/HTTP 三种类型
3. **预设支持** - 支持主流 MCP 服务预设快速配置：
   - GitHub
   - Filesystem
   - Brave Search
   - Slack
   - Memory (Pinecone)
   - Google Maps
   - 等
4. **自定义配置** - 支持自定义 MCP 服务器配置
5. **连接测试** - 配置后自动测试连接
6. **启用/禁用** - 可单独启用/禁用每个服务器
7. **OAuth 支持** - 支持 OAuth token 流程（client_credentials、refresh_token）

#### 2.3.3 MCP 预设列表

| 预设名称 | 图标 | 类型 | 说明 |
|----------|------|------|------|
| GitHub | 🐙 | stdio | GitHub API 集成 |
| Filesystem | 📁 | stdio | 本地文件系统访问 |
| Brave Search | 🔍 | http | 网页搜索 |
| Slack | 💬 | http | Slack 消息集成 |
| Memory (Pinecone) | 🧠 | http | 向量记忆存储 |
| Google Maps | 🗺️ | http | 地图和位置服务 |
| Sentry | 🚨 | http | 错误监控集成 |
| AWS KB | ☁️ | http | AWS 知识库 |

#### 2.3.4 OAuth 配置支持

对于需要 OAuth 的 MCP 服务器（如某些 HTTP/SSE 类型），支持以下配置：
- `client_credentials` 模式
- `refresh_token` 模式
- Token URL 配置
- Client ID/Secret 配置

## 3. 技术方案

### 3.1 前端架构

#### 3.1.1 页面结构

```
frontend/src/app/workspace/
├── settings/
│   ├── page.tsx
│   ├── skills/page.tsx          ← 已有
│   ├── knowledge/page.tsx       ← 新增（用户知识库）
│   └── tasks/page.tsx           ← 新增（用户任务）
├── admin/
│   └── models/
│       ├── mcp/
│       │   ├── page.tsx          ← 已有
│       │   ├── servers/          ← MCP 服务器管理
│       │   └── channels/         ← IM 渠道管理
│       └── ...
```

#### 3.1.2 API 设计

**用户知识库 API**:
```bash
GET    /api/user/knowledge                    # 列出当前用户知识库
POST   /api/user/knowledge                    # 创建知识库
GET    /api/user/knowledge/{id}               # 获取知识库
PUT    /api/user/knowledge/{id}               # 更新知识库
DELETE /api/user/knowledge/{id}               # 删除知识库
POST   /api/user/knowledge/{id}/documents     # 上传文档
DELETE /api/user/knowledge/{id}/documents/{doc_id}  # 删除文档
GET    /api/user/knowledge/{id}/search        # 搜索
POST   /api/user/knowledge/{id}/share         # 分享到工作空间
```

**用户任务 API**:
```bash
GET    /api/user/tasks                         # 列出当前用户任务
POST   /api/user/tasks                         # 创建任务
GET    /api/user/tasks/{id}                    # 获取任务
PUT    /api/user/tasks/{id}                    # 更新任务
DELETE /api/user/tasks/{id}                    # 删除任务
POST   /api/user/tasks/{id}/run                # 手动执行
POST   /api/user/tasks/{id}/pause              # 暂停
POST   /api/user/tasks/{id}/resume             # 恢复
GET    /api/user/tasks/{id}/executions          # 执行历史
POST   /api/user/tasks/{id}/share              # 分享到工作空间
```

**Admin 知识库 API**:
```bash
GET    /api/admin/knowledge                     # 列出所有知识库
PUT    /api/admin/knowledge/{id}                # 更新（可见性等）
DELETE /api/admin/knowledge/{id}                 # 删除
```

**Admin 任务 API**:
```bash
GET    /api/admin/tasks                         # 列出所有任务
PUT    /api/admin/tasks/{id}                    # 更新
DELETE /api/admin/tasks/{id}                    # 删除
POST   /api/admin/tasks/{id}/run                # 强制执行
POST   /api/admin/tasks/{id}/pause              # 暂停
POST   /api/admin/tasks/{id}/resume             # 恢复
```

### 3.2 后端架构

#### 3.2.1 数据模型

class ScheduledTask:
    id: str
    name: str

    cron: str | None
    interval_seconds: int | None
    enabled: bool
    owner_id: str
    workspace_id: str | None
    visibility: str
    shared_workspaces: list[str]
    last_run_at: datetime | None
    last_run_status: str | None
    created_at: datetime
    updated_at: datetime
```

#### 3.2.2 路由结构

```
backend/app/gateway/routers/
├── user_knowledge.py     # 用户知识库 CRUD

## 5. 典型使用场景

### 场景 1: 用户创建知识库并分享

```
1. 用户进入 /workspace/settings/knowledge
2. 点击"创建知识库"，填写名称和描述
3. 上传 PDF/DOCX 文件
4. 测试语义检索
5. 点击"分享"，选择目标工作空间
6. 目标工作空间成员可以在自己的知识库列表中看到共享的知识库
```

### 场景 2: 管理员查看所有用户任务

```
1. 管理员进入 /workspace/admin
2. 点击"定时任务管理"
3. 可以看到所有用户创建的任务列表
4. 可以按用户筛选、搜索
5. 可以强制执行、暂停任何任务
6. 可以修改任务的可见性设置
```

### 场景 3: 用户配置飞书渠道

```
1. 管理员进入 /workspace/admin/models/mcp/channels
2. 点击飞书卡片
3. 填写 App ID 和 App Secret
4. 点击"测试连接"
5. 连接成功后点击"保存"
6. 飞书渠道状态变为"已连接"
```

## 7. 开放问题

| 问题 | 选项 | 建议 |
|------|------|------|
| 知识库/任务入口位置 | 设置页 vs 更多页 | 放置在设置页，与技能管理并列 |
| 分享后权限继承 | 只读 vs 可执行任务 | 知识库只读，任务可执行（分享的是配置，实际执行在创建者上下文） |
| Admin 是否可编辑用户内容 | 是 vs 否 | Admin 可强制停止/删除，但不应修改内容 |
