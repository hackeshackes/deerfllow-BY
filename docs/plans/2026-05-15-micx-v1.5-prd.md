# MicX v1.5 PRD - 功能增强版

**版本目标**：用户级资源管理 + MCP/IM 图形化配置

**日期**：2026-05-15

---

## 一、需求背景

当前 MicX v1.3 已完成定时任务系统和 RAG 知识库的基础功能，但存在以下体验问题：

| 问题 | 当前状态 | 改进方向 |
|------|---------|---------|
| 知识库/定时任务入口 | 仅 Admin 可访问 | 用户工作区可访问 |
| 资源隔离 | 无工作区维度隔离 | 按工作区独立管理 |
| 分享能力 | 不支持跨工作区分享 | 可分享至其他工作区 |
| 权限管理 | Admin 单一视角 | 工作区权限体系 |
| 技能管理 | 不可编辑/删除 | 用户可管理自己的技能 |
| MCP 配置 | 仅 STDIO/SSE/HTTP | 增加主流服务图形化配置 |
| IM 渠道 | YAML 配置 | UI 图形化配置 |

---

## 二、功能规划

### 2.1 用户工作区内的知识库/定时任务

#### 2.1.1 功能描述

用户在工作区内直接管理自己的知识库和定时任务，支持：
- 创建、编辑、删除知识库和定时任务
- 按工作区隔离，数据仅本工作区可见
- 可分享至其他工作区（类似线程的可见性机制）
- 管理员可查看/管理所有用户的资源

#### 2.1.2 导航位置

| 资源 | 建议位置 | 理由 |
|------|---------|------|
| 定时任务 | `/workspace/settings/tasks` 或 `/workspace/more/tasks` | 设置或更多菜单 |
| 知识库 | `/workspace/settings/knowledge` 或 `/workspace/more/knowledge` | 同上 |

#### 2.1.3 数据模型

```typescript
// 定时任务
interface ScheduledTask {
  id: string;
  user_id: string;
  workspace_id: string;          // 所属工作区
  visible_to_workspaces: string[]; // 分享至哪些工作区
  title: string;
  prompt_template: string;
  schedule_type: "cron" | "interval" | "once";
  cron_expression?: string;
  interval_seconds?: number;
  run_at?: string;              // ISO datetime for "once"
  enabled: boolean;
  last_run_at?: string;
  last_run_status?: "success" | "failed";
  created_at: string;
  updated_at: string;
}

// 知识库
interface KnowledgeBase {
  id: string;
  user_id: string;
  workspace_id: string;
  visible_to_workspaces: string[];
  name: string;
  description?: string;
  visibility: "private" | "workspace" | "global";
  document_count: number;
  created_at: string;
  updated_at: string;
}
```

#### 2.1.4 权限规则

| 操作 | 创建者 | 工作区成员 | Admin |
|------|--------|----------|-------|
| 查看自己的资源 | ✅ | N/A | ✅ |
| 编辑自己的资源 | ✅ | N/A | ✅ |
| 删除自己的资源 | ✅ | N/A | ✅ |
| 分享至其他工作区 | ✅ | N/A | ✅ |
| 查看工作区内所有成员资源 | ❌ | ✅ (如果是 shared) | ✅ |
| 管理所有用户资源 | ❌ | ❌ | ✅ |

### 2.2 用户技能管理

#### 2.2.1 功能描述

用户可管理自己创建的技能（远程安装的技能仅 Admin 可操作）：
- 查看自己安装的技能列表
- 编辑技能元数据（名称、描述、图标）
- 删除自己安装的技能

#### 2.2.2 数据模型

```typescript
interface UserSkill {
  id: string;
  name: string;
  display_name: string;
  description: string;
  author: string;          // 作者 ID
  is_builtin: boolean;
  installed_at: string;
  enabled: boolean;
}
```

#### 2.2.3 权限规则

| 技能类型 | 可查看 | 可编辑 | 可删除 |
|---------|--------|--------|--------|
| 内置技能 | ✅ | ❌ | ❌ |
| 远程安装（Admin） | ✅ | ✅ | ✅ |
| 远程安装（用户创建） | ✅ | ✅ | ✅ |

### 2.3 MCP 配置增强

#### 2.3.1 功能描述

在现有 MCP 配置基础上，增加：
- 主流 MCP 服务的预设配置（Filesystem, GitHub, Slack 等）
- 可视化配置界面（表单填写而非直接编辑 JSON）
- 导入/导出配置功能

#### 2.3.2 预设列表（目标）

| 预设名称 | 图标 | 类型 | 描述 |
|---------|------|------|------|
| Filesystem | 📁 | stdio | 本地文件系统访问 |
| GitHub | 🐙 | stdio | GitHub API 集成 |
| Slack | 💬 | stdio | Slack 消息集成 |
| Puppeteer | 🌐 | stdio | 浏览器自动化 |
| Brave Search | 🔍 | http | 网页搜索 |
| Fetch | 🌐 | http | HTTP 请求 |

#### 2.3.3 配置表单

针对每个预设，提供图形化配置表单：

**Filesystem 示例**：
- 工作目录路径（可选，默认用户目录）
- 允许的操作（read/write/delete）

**GitHub 示例**：
- 仓库路径（owner/repo）
- GitHub Token（环境变量引用）

### 2.4 IM 渠道 UI 配置

#### 2.4.1 功能描述

将 YAML 配置转为图形化 UI 配置，放在 MCP 菜单下：

| 渠道 | 配置字段 | 说明 |
|------|----------|------|
| 飞书 | app_id, app_secret, domain | 字节跳动飞书平台 |
| Slack | bot_token, app_token, allowed_users | Socket Mode |
| Telegram | bot_token, allowed_users | Bot API |
| 企业微信 | bot_id, bot_secret | 腾讯企业微信 |
| 钉钉 | client_id, client_secret | 阿里巴巴钉钉 |

#### 2.4.2 配置界面

每个渠道提供独立的配置面板：
- 启用/禁用开关
- 渠道特定配置字段
- 连接测试按钮
- 渠道状态指示

#### 2.4.3 渠道级别 Session 配置

支持按渠道覆盖默认 session 配置：
```yaml
channels:
  telegram:
    enabled: true
    session:
      assistant_id: mobile-agent
      config:
        recursion_limit: 150
```

## 三、技术方案

## 七、关键决策点

### Q1: 知识库/定时任务入口位置

**选项 A**：放在 `/workspace/settings/` 下
- 优点：用户习惯性查找
- 缺点：入口较深

**选项 B**：放在侧边栏"更多"菜单
- 优点：更快速访问
- 缺点：需要用户知道去"更多"找

**推荐**：选项 A（Settings），在侧边栏增加"设置"快捷入口

### Q2: 工作区资源隔离粒度

**选项 A**：严格隔离（资源只能在本工作区查看）
- 适合：企业场景，数据安全要求高

**选项 B**：允许分享（资源可分享至其他工作区）
- 适合：协作场景

**推荐**：选项 B，与线程可见性机制保持一致

### Q3: MCP 预设数量

**选项 A**：仅包含开源/免费服务（Filesystem, Git, Brave Search）
- 优点：无额外成本
- 缺点：功能有限

**选项 B**：包含主流服务（含 Slack, Notion 等）
- 优点：功能完整
- 缺点：需要用户有对应账号

**推荐**：选项 B，预设免费服务，可选付费服务作为扩展
