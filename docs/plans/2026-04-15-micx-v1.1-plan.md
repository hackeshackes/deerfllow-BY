# MicX v1.1 开发计划

## 版本目标

**主题**: *配置中心 UI 化 + 用户技能管理 + 文档完善*

## 一、Admin 配置中心 UI 化

### 1.1 配置项梳理

#### A. 系统配置 (System Config)

| 配置项 | 类型 | 说明 | UI 组件 |
|--------|------|------|---------|
| `log_level` | enum | 日志级别 (debug/info/warning/error) | Select dropdown |
| `token_usage.enabled` | bool | Token 使用统计开关 | Toggle switch |
| `checkpointer.type` | enum | 持久化类型 (memory/sqlite/postgres) | Select dropdown |
| `checkpointer.connection_string` | string | 数据库连接字符串 | Input + Test button |

#### B. 模型配置 (Models Config)

| 配置项 | 类型 | 说明 | UI 组件 |
|--------|------|------|---------|
| `models[].name` | string | 模型名称 | Input |
| `models[].display_name` | string | 显示名称 | Input |
| `models[].use` | string | 模型类型 (langchain_openai:ChatOpenAI 等) | Select |
| `models[].model` | string | 模型名称 (gpt-4o-mini 等) | Input |
| `models[].api_key` | string | API Key (加密存储) | Password input + reveal |
| `models[].base_url` | string | API Base URL (可选) | Input |
| `models[].request_timeout` | float | 请求超时 (秒) | Number input |
| `models[].max_retries` | int | 最大重试次数 | Number input |
| `models[].max_tokens` | int | 最大 token 数 | Number input |
| `models[].temperature` | float | 温度参数 | Slider (0-2) |
| `models[].supports_vision` | bool | 是否支持视觉 | Toggle |

#### C. 沙箱配置 (Sandbox Config)

| 配置项 | 类型 | 说明 | UI 组件 |
|--------|------|------|---------|
| `sandbox.use` | enum | 沙箱类型 (Local/Docker/AIO) | Select |
| `sandbox.allow_host_bash` | bool | 允许主机 bash | Toggle |
| `sandbox.timeout_seconds` | int | 超时秒数 | Number input |

#### D. 工具配置 (Tools Config)

| 配置项 | 类型 | 说明 | UI 组件 |
|--------|------|------|---------|
| `tools[].name` | string | 工具名称 | Input |
| `tools[].group` | string | 工具分组 | Input |
| `tools[].use` | string | 工具实现 | Input |
| `tools[].enabled` | bool | 是否启用 | Toggle |

#### E. 上传配置 (Upload Config)

| 配置项 | 类型 | 说明 | UI 组件 |
|--------|------|------|---------|
| `upload.max_size_mb` | int | 最大上传大小 (MB) | Number input |
| `upload.allowed_extensions` | list | 允许的扩展名 | Multi-select |
| `upload.convert_to_markdown` | bool | 自动转 Markdown | Toggle |

#### F. 技能配置 (Skills Config)

| 配置项 | 类型 | 说明 | UI 组件 |
|--------|------|------|---------|
| `skills.path` | string | 技能目录路径 | Input (readonly) |
| `skills.container_path` | string | 容器内路径 | Input (readonly) |
| `skills.auto_update` | bool | 自动更新技能 | Toggle |
| `skills.security_scan` | bool | 上传前安全扫描 | Toggle |

#### G. MCP 配置 (MCP Config)

| 配置项 | 类型 | 说明 | UI 组件 |

#### 数据模型

```python
class UserSkillConfig(BaseModel):
    skill_name: str
    user_id: str
    enabled: bool = True
    config: dict[str, Any] = {}  # 技能特定参数
    is_default: bool = False
    created_at: datetime
    updated_at: datetime

class SkillShare(BaseModel):
    skill_name: str
    owner_id: str
    workspace_id: str | None  # None = 全局共享
    visibility: str  # "private" | "workspace" | "public"
    created_at: datetime

class SkillRating(BaseModel):
    skill_name: str
    user_id: str
    rating: int  # 1-5
    comment: str | None
    created_at: datetime
```

## 五、技术方案详细设计

### 5.1 配置存储设计

```python
# 现状: config.yaml 文件
# 问题: 每次修改需要重启服务

# 方案: 配置分层存储
config.yaml           # 基础配置 (只读)
+ admin_config.json    # Admin UI 配置 (可写)
+ user_config.json    # 用户配置 (每个用户)
```

### 5.2 配置优先级

```
运行时配置 (最高)
  > Admin UI 配置
    > 用户配置
      > config.yaml (最低)
```

### 5.3 技能存储设计

```
skills/
├── public/           # 内置技能 (只读)
│   ├── skill-a/
│   └── skill-b/
├── shared/           # 共享技能 (工作区级别)
│   └── workspace-{id}/
│       └── skill-c/
└── custom/          # 用户私有技能
    └── user-{id}/
        └── skill-d/
```

### 5.4 API 限流

对用户技能上传添加限制：
- 每个用户最多 10 个私有技能
- 技能包大小限制 5MB
- 上传频率限制 5/分钟
