# MicX 用户端体验路线图 - v1.4.4 起

**版本起点**: v1.4.4  
**主题**: 从“功能集合”升级为“个人 + 团队 AI 工作台”  
**日期**: 2026-04-26

---

## 一、判断

MicX 现在不是缺功能。对话、知识库、定时任务、技能、工作区、共享权限都已经存在。

真正的问题是：**用户不知道这些能力应该怎么组合使用。**

当前产品更像一组功能入口：Chats、Knowledge、Tasks、Skills、Settings、Admin。用户进入后要自己理解：

- 这个对话属于我，还是属于团队？
- 这个知识库会不会被别人看到？
- 定时任务跑出来的结果去哪？
- 技能是我手动选，还是系统自动用？
- 我在个人空间和共享空间之间切换后，到底影响了哪些内容？

v1.4.4 不应该继续堆“置顶、归档、文件夹”这类局部功能。那些功能有用，但不是现在最大的杠杆。

v1.4.4 应该先建立清晰的用户心智：

> 我在个人空间做私人思考。  
> 我在共享空间做团队协作。  
> 对话、知识库、任务、技能都跟随当前空间。  
> 需要共享时显式发布，不默认泄漏。

---

## 二、当前用户端真实痛点

### 2.1 空间心智不清

当前已有工作区切换，但入口藏在左下角设置菜单里。用户不容易意识到“当前空间”是全局上下文。

现状：

- `/workspace/layout.tsx` 已经传入 `activeWorkspaceName`
- `workspace-nav-menu.tsx` 显示“当前空间”，并能切换个人空间/共享空间
- `recent-chat-list.tsx` 只在对话条目上显示“私有 / 已共享” badge
- `knowledge/page.tsx` 用 tabs 区分我的知识库和 workspace knowledge

问题：

- 空间切换不像一级产品动作，更像设置项
- 用户不知道切换空间会影响对话、知识库、任务、技能哪些东西
- “私有 / 已共享”只是属性，不是清晰的信息架构

结论：**个人空间和共享空间应该分开，但不是做两个产品，而是在同一个 workspace shell 里做一级切换和清晰分区。**

---

### 2.2 对话列表混杂

当前对话列表是一个线性 Recent Chat List。

问题：

- 我的私人对话、共享对话、IM 渠道同步来的对话混在一起
- 用户很难判断“这个对话其他人能不能看到”
- 对话多后，最近列表变成垃圾堆
- 对话和任务之间有关联，但用户看不出哪个对话是一次性讨论，哪个是长期自动化任务的结果

用户真正需要的是：

1. **我自己的对话**：私人思考、草稿、实验
2. **共享空间对话**：团队可见、可沉淀、可协作
3. **自动任务对话**：定时运行产生的结果
4. **外部渠道对话**：飞书/Slack/Telegram 进入的消息线程

这四类不应该简单混在 Recent Chats 里。

---

### 2.3 知识库入口重复，使用链路断

当前存在：

- `/workspace/knowledge`
- `/workspace/settings/knowledge`
- admin knowledge page

问题：

- 用户不知道应该去哪个入口管理知识库
- 知识库更像“数据库管理”，不是“AI 使用的资料来源”
- 对话里不能自然选择“这次回答要参考哪些知识库”
- AI 回答后，不能顺手把有价值的内容沉淀回知识库

用户真正想做的是：

- “这次对话请参考公司制度库”
- “把这个结论存进项目知识库”
- “我上传的文件以后还要用，不要每次重新上传”

所以知识库应该从“管理页面”变成“对话上下文的一部分”。

---

### 2.4 定时任务不像用户工作流

当前定时任务已经显示在侧边栏，也有运行、暂停、恢复。

问题：

- 用户看到的是“任务对象”，不是“自动帮我做的事”
- 任务结果如果写入 thread，入口不够清楚
- 任务缺少下一次运行、上次结果、失败原因、结果投递位置的统一视图
- 用户不容易从一次对话自然升级为定时任务

用户真正想做的是：

- “每天早上帮我看飞书未读和日程，生成简报”
- “每周五总结这个项目空间的进展”
- “每次知识库更新后，提醒我有哪些重要变化”

所以定时任务应该改名和重构为：**自动化 / Automations**。

---

### 2.5 技能管理偏开发者，不偏用户

当前用户技能页有：全部技能、我的创建、共享记录。

问题：

- 普通用户不理解“技能管理”的价值
- 用户关心的是“我能让 AI 做什么”，不是“enable/disable 哪个 skill”
- 创建技能需要写 SKILL.md 模板，对普通用户太硬
- 技能没有和新建对话、任务、知识库场景自然连接

用户真正需要的是：

- “帮我写周报”的模板
- “帮我审合同”的流程
- “每周自动跑一次项目总结”的自动化能力
- “这个团队共享了哪些好用的工作流”

所以技能应该在用户侧包装为：**工作流模板 / AI 能力卡片**。

## 三、产品原则

### 3.1 空间优先

所有用户对象都要回答一个问题：它属于哪里？

| 对象 | 个人空间 | 共享空间 |
|------|----------|----------|
| 对话 | 默认私有 | 默认空间成员可见 |
| 知识库 | 仅自己可检索 | 空间成员可检索 |
| 自动化 | 仅自己收到结果 | 输出到共享对话/频道 |
| 技能/模板 | 自己可用 | 空间成员可用 |
| 记忆 | 个人偏好 | 团队约定和项目背景 |

### 3.2 共享必须显式

个人空间内容默认不进入共享空间。用户主动点击“发布到共享空间”后才共享。

### 3.3 对话是核心，不是孤岛

对话应该能自然连接：知识库、任务、技能、文件、外部渠道。

### 3.4 少做管理页，多做上下文动作

用户不想“管理系统”。用户想在当前场景里完成动作。

例如：

- 在对话里选择知识库
- 在 AI 回复下方存入知识库
- 从一段对话创建自动化
- 从新建页选择工作流模板

## 四、v1.4.4 推荐范围：空间化工作台

v1.4.4 的目标不是做完整大版本，而是建立正确的信息架构。

### 4.1 一级空间切换

把“个人空间 / 共享空间”从设置菜单提升到左侧边栏顶部。

设计：

```text
┌────────────────────────┐
│ MicX                   │
│ [个人空间 ▼]            │  ← 一级切换
├────────────────────────┤
│ + 新建对话              │
│ 对话                    │
│ 知识库                  │
│ 自动化                  │
│ 工作流                  │
└────────────────────────┘
```

切换后，下面所有列表都跟随空间过滤。

验收：

- 用户能一眼看到当前空间
- 切换空间后，对话/知识库/自动化/工作流都刷新到该空间
- 个人空间有“发布到共享空间”动作
- 共享空间有“复制到个人空间”动作

---

### 4.2 对话分区，而不是简单文件夹

左侧 Recent Chats 改为按空间和来源分区。

个人空间：

```text
我的对话
├── 最近继续
├── 草稿与实验
└── 已发布到共享空间
```

共享空间：

```text
共享对话
├── 团队最近讨论
├── 自动化结果
└── 外部渠道对话
```

先不做复杂文件夹。文件夹是后续增强，不是 v1.4.4 的核心。

新增 metadata：

```text
thread.metadata.space_type = personal | workspace
thread.metadata.source = manual | automation | channel
thread.metadata.published_from_thread_id = optional
thread.metadata.last_actor = user | agent | automation | channel
```

验收：

- 个人空间默认只看到个人对话
- 共享空间默认只看到共享对话
- IM 渠道对话在共享空间中单独分区
- 自动任务生成的 thread 在“自动化结果”中展示

---

### 4.3 新建对话变成“选择场景”

不要只是空白输入框。新建对话页应该先问：你要在哪个空间，用哪些资料，做什么？

设计：

```text
当前空间：产品团队空间

你想做什么？
[自由对话] [基于知识库问答] [写文档] [分析文件] [创建自动化]

可用知识库：
[ ] 产品文档库  [ ] 客户反馈库  [ ] 公司制度库

可用工作流：
[周报总结] [会议纪要] [需求评审] [代码审查]
```

行为：

- 个人空间默认不带共享知识库，除非用户勾选
- 共享空间默认推荐该空间知识库
- 用户选择工作流后，自动填入推荐 prompt 和模式

验收：

- 新建对话时明确显示当前空间
- 可勾选知识库作为本次上下文
- 可选择工作流模板启动
- 选择“创建自动化”进入定时任务创建流程

---

### 4.4 知识库变成“资料来源”

v1.4.4 不做复杂 RAG 技术升级，先改用户心智。

页面名称建议：

- 用户侧：`资料库` 或 `知识来源`
- Admin 侧：仍叫 `知识库管理`

用户侧 tabs：

```text
当前空间资料
我的私人资料
全局资料
```

核心动作：

- 上传资料
- 设为本空间默认资料
- 在对话中引用
- 从 AI 回复存入资料库

验收：

- 资料库页面只展示当前空间相关资料
- 对话输入框能选择资料来源
- AI 回复能一键存入某个资料库
- 用户能看到每个资料库的可见范围

---

### 4.5 定时任务改为“自动化”

用户侧不要叫定时任务。叫“自动化”。

自动化列表按用户能理解的字段展示：

```text
名称：每日晨报
触发：每天 09:00
资料来源：飞书日程 + 项目知识库
输出到：个人对话 / 产品团队空间
上次结果：成功，2分钟前
下次运行：明天 09:00
```

从对话创建自动化：

```text
AI 回复菜单 → 转为自动化
```

验收：

- `/workspace/tasks` 用户侧改名为 `/workspace/automations`，旧路由兼容
- 对话中可以“转为自动化”
- 自动化有个人/共享空间归属
- 自动化结果写回明确的目标对话或空间结果流

---

### 4.6 技能变成“工作流模板”

用户侧弱化“技能管理”，强化“可直接使用”。

用户入口：`工作流`

分类：

- 我的常用
- 当前空间推荐
- 团队共享
- 系统内置

卡片文案不要写技术描述，而写用户任务：

```text
会议纪要
把会议录音/纪要整理成：结论、行动项、负责人、截止时间。

[使用] [设为常用] [创建自动化]
```

验收：

- 普通用户能从工作流卡片直接开始对话
- 工作流能绑定默认模型/模式/知识库
- 工作流能一键转为自动化
- 高级编辑 SKILL.md 仍保留，但放到“高级设置”

---

## 五、v1.4.5 推荐范围：沉淀与复用

v1.4.4 解决“在哪里做事”。v1.4.5 解决“做完以后怎么沉淀”。

### 5.1 对话沉淀

- AI 回复一键保存为资料
- 整个对话生成总结页
- 一键发布为共享空间知识
- 对话结论可生成任务/自动化/文档

### 5.2 资料库质量

- 上传后显示解析状态
- 支持失效/过期资料标记
- 支持资料去重
- 支持引用来源预览

### 5.3 工作流复用

- 从优秀对话生成工作流
- 工作流支持参数：目标、输出格式、语气、资料来源
- 工作流版本管理

---

## 六、v1.4.6 推荐范围：协作闭环

v1.4.6 做团队协作，不做复杂实时协同。

### 6.1 共享对话协作

- 多人可查看同一对话
- 输入串行处理，清晰显示“AI 正在回复谁”
- 每条用户消息显示发送人
- 支持评论/引用某条 AI 回复

### 6.2 共享空间动态流

类似团队空间首页：

```text
今天发生了什么
- 张三发布了一个对话总结
- 每日晨报自动化运行成功
- 产品资料库新增 3 个文档
- 会议纪要工作流被使用 5 次
```

### 6.3 通知与订阅

- 订阅某个自动化结果
- 订阅某个资料库更新
- 订阅某个共享对话的新回复

---

## 七、v1.5 推荐范围：平台能力扩展

原 v1.5 的钉钉、企业微信、Slack 工具扩展仍然有价值，但建议延后到用户端心智建立之后。

否则工具越多，用户越不知道怎么用。

v1.5 应该围绕“外部工作流接入”来做：

- 飞书/钉钉/企微/Slack 的消息、任务、日历能力
- 外部渠道对话进入共享空间
- 自动化结果投递到外部群
- 外部资料同步进空间资料库

---

## 八、优先级建议

### v1.4.4 必做

1. 左侧一级空间切换
2. 对话列表按个人/共享/自动化/渠道分区
3. 新建对话场景化：空间 + 资料来源 + 工作流
4. 知识库改名/重组为资料来源
5. 定时任务用户侧改名为自动化

### v1.4.4 可选

1. AI 回复一键存入知识库
2. 工作流卡片启动对话
3. 对话转自动化

### 暂缓

1. 文件夹/拖拽分组
2. 大而全的全文搜索
3. 复杂实时多人协作
4. 技能 marketplace

---

## 九、为什么这样排

如果先做置顶、归档、文件夹，只是在混乱列表上继续补丁。

如果先做更多平台工具，只会增加复杂度。

如果先做空间化工作台，后面的知识库、自动化、技能、外部渠道都会有自然归属。

这是地基。

---

## 十、验收标准

v1.4.4 结束时，用户应该能清楚回答：

1. 我现在在哪个空间？
2. 这个对话谁能看到？
3. 这次 AI 会参考哪些资料？
4. 这个自动化的结果会发到哪里？
5. 这个工作流是我个人的，还是团队共享的？

如果这 5 个问题不能一眼回答，v1.4.4 就没有真正成功。
