### 1.2 创建 Voice Router

**文件**: `backend/app/gateway/routers/voice.py` (新建)

**内容**:
- `POST /api/voice/stt` - 接收音频文件，调用 faster-whisper 转写
- `GET /api/voice/config` - 获取语音配置（扩展 stt_model_size）
- `PUT /api/voice/config` - 更新语音配置

**核心逻辑**:
```python
@router.post("/stt")
async def speech_to_text(request: Request, file: UploadFile = File(...)):
    # 1. 验证文件格式 (audio/webm)
    # 2. 保存临时文件
    # 3. 加载 faster-whisper 模型（带缓存）
    # 4. 执行转写
    # 5. 删除临时文件
    # 6. 返回 {"text": "...", "language": "zh", "duration": 5.2}
```

**模型加载策略**:
- 首次调用时加载模型到内存
- 模型缓存：使用 `faster_whisper` 的默认缓存目录
- 根据 `stt_model_size` 配置选择模型

**模型下载**:
- tiny: `Systran/faster-whisper-tiny`
- small: `Systran/faster-whisper-small`
- medium: `Systran/faster-whisper-medium`

## 技术决策点（需确认）

### 1. 模型下载策略
- **A**: 首次使用时自动从 HuggingFace Hub 下载
- **B**: 预下载模型到项目目录，Docker 构建时打包

**推荐**: A（更灵活，首次使用自动处理）

### 2. 模型缓存位置
- 使用 `faster-whisper` 默认缓存（`~/.cache/huggingface/`）
- 或指定项目目录缓存（`data/models/whisper/`）

**推荐**: 项目目录缓存（方便清理和管理）

### 3. Admin 页面位置
- **A**: 在现有模型管理页面（models-admin-page.tsx）增加 STT 配置区块
- **B**: 新建独立语音配置页面

**推荐**: A（保持现有结构，减少页面数量）

---

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 模型下载慢/失败 | 功能不可用 | 提供预下载选项，添加超时处理 |
| 模型加载占用大量内存 | 服务不稳定 | 小模型优先，支持配置 |
| 录音格式兼容性问题 | 转写失败 | 前端固定 webm 格式，后端验证 |
