## 一、版本目标与核心价值

### 1.1 版本愿景

v1.4 将 MicX 从**单用户 AI 助手**进化为**企业级多用户协作平台**：

| 现有能力 | v1.4 新增 | 价值跃升 |
|---------|----------|---------|
| 任务/知识库仅创建者可见 | **工作区隔离 + 跨区分享** | 从"个人工具"到"团队资产" |
| 技能仅可管理原内置技能 | **自定义技能完整 CRUD** | 从"平台提供"到"用户创造" |
| MCP 配置分散在配置中心 | **独立 MCP 管理菜单 + 图形化配置** | 从"工程师配置"到"产品化配置" |
| IM 渠道无 UI | **图形化 IM 渠道配置** | 从"手动编辑"到"一键接入" |

### 1.2 核心场景

**场景A - 团队知识协作**
```
管理员: 在工作区A创建知识库 "产品文档"
管理员: 将知识库分享到工作区B
工作区B成员: 可以在自己的知识库中访问产品文档
工作区A成员: 可以管理分享的知识库
```

**场景B - 定时任务团队支持**
```
用户A: 创建任务 "每日销售报告"，分享到工作区
工作区成员: 可以在自己的工作区看到任务执行结果
管理员: 可以管理所有用户的定时任务
```

**场景C - MCP 图形化配置**
```
管理员: 进入模型管理 → MCP 配置
管理员: 点击 "添加 MCP Server"
系统: 显示预设列表（GitHub、Slack、PostgreSQL 等）
管理员: 选择 "GitHub"，填写必要参数
系统: 自动验证连接，启用 MCP Tools
```

**场景D - IM 渠道一键接入**
```
管理员: 进入 MCP 菜单 → IM 渠道
管理员: 点击 "飞书" 渠道
系统: 显示配置表单（App ID、App Secret 等）
管理员: 填写配置，点击启用
系统: 自动启动 WebSocket 连接，渠道在线
```

## 二、功能需求详述

### 2.1 任务与知识库工作区隔离

#### 2.1.1 数据模型变更

**现有 model**:
```python
class ScheduledTask:
    user_id: str  # 创建者

class KnowledgeBase:
    user_id: str  # 创建者
```

**v1.4 model**:
```python
class ScheduledTask:
    id: str
    user_id: str           # 创建者
    workspace_id: str       # 所属工作区
    shared_to: list[str]   # 分享到的工作区列表
    visibility: str         # "private" | "workspace" | "shared"

class KnowledgeBase:
    id: str
    user_id: str           # 创建者
    workspace_id: str       # 所属工作区
    shared_to: list[str]    # 分享到的工作区列表
    visibility: str         # "private" | "workspace" | "shared"
```

#### 2.1.2 权限体系

| 角色 | 我的任务/知识库 | 分享给我的 | 工作区全部 | 所有用户（管理员） |
|------|----------------|----------|-----------|------------------|
| 成员 | CRUD + 分享 | 查看 + 使用 | - | - |
| 空间管理员 | CRUD + 分享 | 查看 + 使用 | 查看 + 管理 | - |
| 系统管理员 | CRUD + 分享 | 查看 + 使用 | 查看 + 管理 | 全部管理 |

#### 2.1.3 用户界面变更

**任务列表页 (`/workspace/tasks`)**:
- Tab 切换: `我的任务` | `工作区任务` | `分享的任务`
- 每个任务卡片显示: 分享状态图标、所属工作区
- 新增 "分享" 按钮 → 分享到工作区弹窗

**知识库列表页 (`/workspace/knowledge`)**:
- Tab 切换: `我的知识库` | `工作区知识库` | `分享的知识库`
- 每个知识库卡片显示: 分享状态图标、来源工作区

#### 2.1.4 分享流程

```
用户点击 "分享" → 选择工作区 → 设置权限（只读/可编辑）→ 确认
→ 系统创建分享记录 → 目标工作区成员可见
```

#### 2.1.5 管理员视图

**管理员菜单** (`/workspace/admin/tasks`, `/workspace/admin/knowledge`):
- 下拉筛选: `全部用户` | `指定用户`
- 表格展示: 用户名、工作区、名称、类型、分享状态、操作
- 支持: 禁用任务/知识库、删除、查看详情

### 2.2 自定义技能完整 CRUD

#### 2.2.1 技能分类

| 类型 | 来源 | 可见范围 | 管理权限 |
|------|------|---------|---------|
| 内置技能 | 系统预装 | 全局 | 仅管理员 |
| 自定义技能 | 用户创建 | 创建者/指定工作区 | 创建者/管理员 |

#### 2.2.2 用户技能管理

**用户操作**:
- 创建技能: `/workspace/settings/skills` → "创建自定义技能"
- 编辑技能: 点击技能卡片 → 进入编辑页面
- 删除技能: 点击删除 → 确认弹窗
- 分享技能: 点击分享 → 选择工作区

**技能创建/编辑表单**:
```yaml
name: string          # 技能名称（唯一）
display_name: string   # 显示名称
description: string    # 描述
content: string        # SKILL.md 内容
visibility: enum       # private | workspace
workspace_id: string   # 分享到的工作区（可选）
```

#### 2.2.3 权限控制

- 用户只能编辑/删除自己创建的技能
- 管理员可以编辑/删除所有自定义技能
- 内置技能不可删除，仅管理员可禁用

## 三、技术方案

### 3.1 任务与知识库工作区隔离

#### 3.1.1 数据库变更

**新增表**:
```sql
-- 任务分享表
CREATE TABLE task_shares (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    target_workspace_id TEXT NOT NULL,
    permission TEXT DEFAULT 'read',  -- 'read' | 'write'
    shared_by TEXT NOT NULL,
    shared_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (task_id) REFERENCES scheduled_tasks(id),
    FOREIGN KEY (target_workspace_id) REFERENCES workspaces(id)
);

-- 知识库分享表
CREATE TABLE knowledge_shares (
    id TEXT PRIMARY KEY,
    knowledge_base_id TEXT NOT NULL,
    target_workspace_id TEXT NOT NULL,
    permission TEXT DEFAULT 'read',
    shared_by TEXT NOT NULL,
    shared_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (knowledge_base_id) REFERENCES knowledge_bases(id),
    FOREIGN KEY (target_workspace_id) REFERENCES workspaces(id)
);
```

**变更表**:
```sql
ALTER TABLE scheduled_tasks ADD COLUMN workspace_id TEXT;
ALTER TABLE scheduled_tasks ADD COLUMN visibility TEXT DEFAULT 'private';
ALTER TABLE knowledge_bases ADD COLUMN workspace_id TEXT;
ALTER TABLE knowledge_bases ADD COLUMN visibility TEXT DEFAULT 'private';
```

#### 3.1.2 API 变更

**任务 API**:
```
GET    /api/tasks                    # 返回用户有权访问的任务（含分享）
POST   /api/tasks                    # 创建时指定 workspace_id
GET    /api/tasks/{id}               # 检查访问权限
POST   /api/tasks/{id}/share         # 分享到工作区
DELETE /api/tasks/{id}/share/{ws_id} # 取消分享
GET    /api/admin/tasks              # 管理员: 所有任务
```

**知识库 API**:
```
GET    /api/knowledge                 # 返回用户有权访问的知识库
POST   /api/knowledge                # 创建时指定 workspace_id
GET    /api/knowledge/{id}           # 检查访问权限
POST   /api/knowledge/{id}/share     # 分享到工作区
DELETE /api/knowledge/{id}/share/{ws_id}
GET    /api/admin/knowledge          # 管理员: 所有知识库
```

#### 3.1.3 分享权限检查

```python
def can_access_task(user_id: str, task_id: str, workspace_id: str) -> bool:
    task = get_task(task_id)
    if task.user_id == user_id:
        return True
    if task.workspace_id == workspace_id and task.visibility in ("workspace", "shared"):
        return True
    if task.shared_to contains workspace_id:
        return True
    return False
```
