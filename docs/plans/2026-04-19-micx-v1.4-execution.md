### 1.2 数据库模型

#### 1.2.1 新增表

**task_shares**:
```sql
CREATE TABLE task_shares (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    target_workspace_id TEXT NOT NULL,
    permission TEXT DEFAULT 'read',
    shared_by TEXT NOT NULL,
    shared_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (task_id) REFERENCES scheduled_tasks(id),
    FOREIGN KEY (target_workspace_id) REFERENCES workspaces(id)
);
```

**knowledge_shares**:
```sql
CREATE TABLE knowledge_shares (
    id TEXT PRIMARY KEY,
    knowledge_base_id TEXT NOT NULL,
    target_workspace_id TEXT NOT NULL,
    permission TEXT DEFAULT 'read',
    shared_by TEXT NOT NULL,
    shared_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (knowledge_base_id) REFERENCES knowledge_bases(id),
    FOREIGN KEY (target_workspace_id) REFERENCES workspaces(id)
);
```

**custom_skills**:
```sql
CREATE TABLE custom_skills (
    id TEXT PRIMARY KEY,
    name TEXT UNIQUE NOT NULL,
    display_name TEXT NOT NULL,
    description TEXT,
    content TEXT NOT NULL,
    author_id TEXT NOT NULL,
    visibility TEXT DEFAULT 'private',
    workspace_id TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (author_id) REFERENCES users(id),
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id)
);
```

#### 1.2.2 变更表

**scheduled_tasks**:
```sql
ALTER TABLE scheduled_tasks ADD COLUMN workspace_id TEXT;
ALTER TABLE scheduled_tasks ADD COLUMN visibility TEXT DEFAULT 'private';
```

**knowledge_bases**:
```sql
ALTER TABLE knowledge_bases ADD COLUMN workspace_id TEXT;
ALTER TABLE knowledge_bases ADD COLUMN visibility TEXT DEFAULT 'private';
```

### 1.3 API 端点

#### 1.3.1 任务分享

```
POST   /api/tasks/{id}/share
       Body: { target_workspace_id, permission }
       Response: { share_id, task_id, target_workspace_id }

DELETE /api/tasks/{id}/share/{share_id}
       Response: { success: true }

GET    /api/admin/tasks
       Query: ?user_id=&workspace_id=&page=&limit=
       Response: { tasks: [...], total }
```

#### 1.3.2 知识库分享

```
POST   /api/knowledge/{id}/share
DELETE /api/knowledge/{id}/share/{share_id}
GET    /api/admin/knowledge
```

#### 1.3.3 自定义技能 CRUD

```
GET    /api/user/skills/custom
POST   /api/user/skills/custom
GET    /api/user/skills/custom/{name}
PUT    /api/user/skills/custom/{name}
DELETE /api/user/skills/custom/{name}
POST   /api/user/skills/custom/{name}/share
```

#### 1.3.4 MCP 配置

```
GET    /api/mcp/presets
       Response: { presets: [...] }

POST   /api/mcp/servers
       Body: { preset_id, config }
       Response: { server_id, status }

GET    /api/mcp/servers
DELETE /api/mcp/servers/{id}
PUT    /api/mcp/servers/{id}
POST   /api/mcp/servers/{id}/test

GET    /api/mcp/servers/{id}/status
       Response: { connected: bool, error?: string }
```

#### 1.3.5 IM 渠道配置

```
GET    /api/channels/config
       Response: { channels: [...] }

PUT    /api/channels/config/{channel_type}
       Body: { enabled, config }
       Response: { status }

GET    /api/channels/status
       Response: { telegram: {...}, slack: {...}, feishu: {...}, wecom: {...} }

POST   /api/channels/{channel_type}/start
POST   /api/channels/{channel_type}/stop
```

## 六、代码规范

### 6.1 Backend

- 遵循现有 `ruff.toml` 配置
- 新增 API 必须有 OpenAPI 文档注释
- 所有数据库操作使用 async
- 错误处理使用 try-except + 日志记录

- 遵循现有 ESLint/Prettier 配置
- 组件放在 `components/workspace/` 对应子目录
- API 调用通过 `core/api/` 封装
- i18n key 放在 `src/core/i18n/locales/` 对应文件
