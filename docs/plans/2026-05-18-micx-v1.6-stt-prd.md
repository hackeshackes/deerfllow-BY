# MicX v1.6 PRD：语音转文字（STT）功能 + Admin 模型大小配置

## 1. 概述与目标

### 版本目标
为 MicX 增加**内置语音转文字（STT）功能**，基于 faster-whisper 实现完全离线的语音识别能力。同时在 Admin 后台模型管理中增加 **STT 模型大小配置开关**（小/中/大），供管理员根据服务器资源情况选择合适的模型。

### 解决的问题
- 用户点击麦克风按钮后，录音正常但无法完成语音转文字（后端 API 不存在）
- 现有代码 `voice-input.tsx` 调用 `/api/voice/stt` 返回 404

### 核心价值
- **隐私安全**：语音数据全程在本地处理，不上传第三方
- **离线可用**：无需外网连接，无 API 成本
- **灵活配置**：管理员可根据服务器配置选择模型大小

## 2. 功能详情

### 2.1 STT 后端实现

#### 技术方案
- **引擎**：faster-whisper（CTranslate2 优化的 OpenAI Whisper）
- **模型来源**：HuggingFace Hub 自动下载（首次使用时）
- **中文支持**：Whisper 训练数据包含大量中文，识别效果好

#### 模型规格
| 规格 | 模型 | 内存占用 | 速度 | 适用场景 |
|------|------|----------|------|----------|
| **小（tiny）** | faster-whisper tiny | ~500MB | 极快 | CPU 服务器、资源受限环境 |
| **中（small）** | faster-whisper small | ~1GB | 快 | 平衡场景，推荐 |
| **大（medium）** | faster-whisper medium | ~2.5GB | 较慢 | 高质量需求、有 GPU |

#### API 设计

**1. 语音转文字**
```
POST /api/voice/stt
Content-Type: multipart/form-data

file: audio/webm (录音文件)

Response 200:
{
  "text": "转录文字内容",
  "language": "zh",
  "duration": 5.2
}

Response 400: 文件格式错误
Response 500: 转录失败
```

**2. 获取语音配置**
```
GET /api/voice/config

Response 200:
{
  "stt_enabled": true,
  "tts_enabled": true,
  "stt_language": "zh",
  "stt_model_size": "small",
  "tts_voice": "zh-CN-XiaoxiaoNeural",
  "tts_speed": 1.0
}
```

**3. 更新语音配置**
```
PUT /api/voice/config
Content-Type: application/json

{
  "stt_model_size": "small"  // tiny | small | medium
}

Response 200: 更新后的配置
```

### 2.2 Admin 模型大小配置

#### 位置
Admin 后台 → 模型管理 → 新增/编辑模型页面 → **STT 模型大小** 下拉选择

#### 选项
- 小（tiny） - 快速响应，低资源占用
- 中（small） - 平衡模式（**默认**）
- 大（medium） - 高质量，需较强算力

#### 存储
- 存储于 `backend/app/gateway/data/voice_config.py` (已有 SQLite)
- 字段：`stt_model_size: tiny | small | medium`

### 2.3 前端集成

#### 录音按钮
- 现有 `voice-input.tsx` 组件已实现录音逻辑
- 录音完成 → 调用 `/api/voice/stt` → 获取文字 → 填入输入框

#### 权限控制
- 需要用户已登录
- 管理员可配置全局 STT 开/关

## 3. 用户流程

### 语音输入流程
```
用户点击麦克风按钮
    ↓
浏览器请求麦克风权限
    ↓
用户开始说话，实时显示录音波形
    ↓
用户点击停止或超时自动停止
    ↓
前端发送 audio/webm 到 /api/voice/stt
    ↓
后端 faster-whisper 转写
    ↓
返回文字到前端
    ↓
文字填入输入框，用户可编辑后发送
```

### Admin 配置流程
```
管理员登录 → 进入 Admin 控制台
    ↓
进入 模型管理 页面
    ↓
点击 新增/编辑
    ↓
在 STT 模型大小 下拉菜单选择
    ↓
保存配置
    ↓
下次用户使用语音时自动使用新配置
```

## 4. 数据结构

### 语音配置 (SQLite)
```python
voice_configs 表:
- user_id: TEXT PRIMARY KEY
- stt_enabled: INTEGER (0/1)
- tts_enabled: INTEGER (0/1)
- stt_language: TEXT (默认 "zh")
- stt_model_size: TEXT (tiny/small/medium)
- tts_voice: TEXT
- tts_speed: REAL
- created_at: REAL
- updated_at: REAL
```

### API 响应模型
```python
class VoiceSTTResponse(BaseModel):
    text: str
    language: str
    duration: float

class VoiceConfigResponse(BaseModel):
    stt_enabled: bool
    tts_enabled: bool
    stt_language: str
    stt_model_size: str  # 新增
    tts_voice: str
    tts_speed: float

class VoiceConfigUpdate(BaseModel):
    stt_enabled: bool | None = None
    tts_enabled: bool | None = None
    stt_language: str | None = None
    stt_model_size: str | None = None  # tiny/small/medium
    tts_voice: str | None = None
    tts_speed: float | None = None
```

## 5. 非功能性要求

### 性能要求
- STT 响应时间 < 3秒（small 模型，10秒内录音）
- 支持最长 60 秒录音转写

### 资源要求
- 内存：small 模型 ~1GB RAM
- 磁盘：模型缓存约 500MB-2.5GB（根据模型大小）
- 首次使用自动下载模型到缓存目录

### 错误处理
- 录音过短（< 0.5秒）：返回 "录音时间过短"
- 录音过长（> 60秒）：返回 "录音时间过长"
- 无语音内容：返回 "未检测到语音内容"
- 模型加载失败：返回具体错误信息
