# MicX v1.5 PRD - 平台扩展与体验升级

**版本目标**: *扩展平台能力（钉钉/企业微信/群组管理）+ 对话体验升级（标签、搜索增强）+ 工具质量加固*

**日期**: 2026-04-25

---

## 一、背景

v1.4.2 完成了飞书平台 57 个工具的建设，覆盖云文档、多维表格、日历、任务、邮件、表格、云盘、审批、妙记、OKR、考勤等模块。

v1.5 将聚焦三个方向：

1. **平台扩展**：钉钉、企业微信的 API 工具建设（两个渠道配置已存在但无工具）
2. **体验升级**：对话标签管理 + 对话搜索增强
3. **质量加固**：飞书工具深度测试 + Slack 工具补全

| | `dingtalk_calendar_list` | 列出日程 |
| **任务** | `dingtalk_task_add` | 创建任务 |
| | `dingtalk_task_list` | 列出任务 |

**文件结构**:

## 三、体验升级

### 3.1 对话标签

支持在对话（Thread）上打标签，方便管理和检索。

**API 端点**:
```bash
POST   /api/threads/{id}/tags           # 添加标签
DELETE /api/threads/{id}/tags/{tag}     # 删除标签
GET    /api/threads/{id}/tags           # 获取标签列表
GET    /api/threads/tags/{tag}          # 按标签搜索对话
```

**数据模型**:
```python
class ThreadTag(BaseModel):
    id: int
    name: str
    color: str  # 标签颜色
    created_at: datetime
    created_by: str
```

**文件**:
- `backend/app/gateway/routers/threads.py` - 新增标签 CRUD
- `backend/app/gateway/models/thread.py` - 添加 tags 关联
- `frontend/src/components/thread/` - 标签管理 UI

### 3.2 对话搜索增强

当前搜索仅支持标题，v1.5 扩展为全文检索 + 多维度过滤。

**API 端点**:
```bash
GET    /api/threads/search?q=关键词&tags=a,b&workspace_id=x&user_id=y
```

**搜索能力**:
- 对话标题全文检索
- 按标签过滤
- 按工作区过滤
- 按用户过滤
- 按时间范围过滤
- 结果分页

**文件**:
- `backend/app/gateway/routers/threads.py` - 搜索增强
- `frontend/src/components/thread/` - 搜索 UI 升级

### 4.2 工具错误处理标准化

当前各工具错误处理不一致，v1.5 统一错误码和错误格式。

**统一错误格式**:
```json
{
  "error": true,
  "code": "FEISHU_API_ERROR",
  "message": "飞书 API 调用失败",
  "details": {
    "api_code": 99991661,
    "api_msg": "invalid parameter"
  }
}
```

**错误码规范**:
| 错误码 | 说明 |
|--------|------|
| `FEISHU_NOT_CONFIGURED` | 飞书渠道未配置 |
| `FEISHU_API_ERROR` | 飞书 API 调用失败 |
| `FEISHU_PERMISSION_DENIED` | 权限不足 |
| `FEISHU_RATE_LIMITED` | 限流 |
| `INVALID_PARAMETER` | 参数错误 |

## 八、风险点

1. **钉钉/企业微信 SDK**：第三方 SDK 稳定性不如官方，需要充分测试
2. **标签数据库迁移**：需要 ALTER TABLE，对已有数据做兼容
3. **全文检索性能**：对话内容大后搜索可能慢，考虑添加索引
4. **API 限流**：钉钉/企业微信均有调用频率限制
