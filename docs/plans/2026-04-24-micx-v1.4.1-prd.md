## 一、版本目标与核心价值

### 1.1 版本愿景

v1.4.1 将 MicX 技能系统升级为**企业级技能治理平台**,并开始构建**飞书操作能力**：

| 现有能力 | v1.4.1 新增 | 价值跃升 |
|---------|-------------|---------|
| 技能使用固定模型 | **技能可绑定默认模型** | 从"全局模型"到"技能级模型适配" |
| 技能删除无权限控制 | **管理员权限管控** | 从"自由删除"到"权限隔离" |
| 飞书仅支持收发消息 | **飞书消息读取/搜索** | 从"被动接收"到"主动获取" |

### 1.2 核心场景

**场景A - 技能模型适配**
```
管理员: 进入技能管理 → 选择 "image-generation"
管理员: 为其绑定默认模型 "dall-e-3"
用户: 使用 image-generation 技能时,自动使用 dall-e-3
用户: 也可在对话中临时切换为 "gpt-image"
```

**场景B - 管理员技能保护**
```
管理员: 安装的 "公司考勤技能" 标记为管理员技能
普通用户: 可以启用/禁用,但无法删除
管理员: 可以删除或更新
系统内置技能: 所有用户只能查看,无法操作
```

**场景C - 飞书消息搜索**
```
用户: 在对话中输入 "帮我搜索飞书群里的产品反馈"
系统: 调用飞书消息搜索 API
系统: 返回相关消息列表
用户: 选择消息,继续 AI 分析
```

## 二、功能需求详述

### 2.1 技能模型配置中心

#### 2.1.1 模型能力标签

**功能**: 为每个模型标记其支持的能力类型

**能力标签类型**:
| 标签 | 说明 | 示例模型 |
|------|------|----------|
| `text` | 文本生成 | gpt-4o, claude-3.5 |
| `image-gen` | 图像生成 | dall-e-3, midjourney |
| `video-gen` | 视频生成 | sora, runway |
| `code-gen` | 代码生成 | gpt-4o, cursor |
| `vision` | 视觉理解 | gpt-4o, claude-3.5 |
| `thinking` | 推理思考 | o1, claude-3.7 |

**API 模型**:
```python
class ModelCapabilities(BaseModel):
    """模型能力标签"""
    capabilities: list[str] = Field(
        default=["text"],
        description="支持的能力列表: text, image-gen, video-gen, code-gen, vision, thinking"
    )
```

**UI 展示**:
- 模型卡片显示能力标签徽章
- 技能绑定时,只显示兼容的模型

#### 2.1.2 技能模型映射

**功能**: 为技能绑定默认模型

**数据模型**:
```python
class SkillModelBinding(BaseModel):
    """技能-模型绑定"""
    skill_name: str
    model_name: str
    is_override_allowed: bool = True  # 是否允许用户覆盖
    created_by: str
    created_at: datetime
```

**存储方式**: `extensions_config.json`
```json
{
  "skill_model_bindings": {
    "image-generation": {
      "model_name": "dall-e-3",
      "is_override_allowed": true
    },
    "deep-research": {
      "model_name": "gpt-4o",
      "is_override_allowed": false
    }
  }
}
```

**UI 流程**:
1. Admin 进入 `/workspace/admin/models/skills`
2. 选择某个技能
3. 点击 "绑定模型" → 显示兼容模型列表
4. 选择模型,设置是否允许覆盖
5. 保存

#### 2.1.3 运行时模型选择

**功能**: 技能执行时按配置自动选择模型

**执行流程**:
```
用户: 在对话中使用技能 "image-generation"
系统: 检查技能是否绑定模型
  ├─ 已绑定 → 使用绑定模型 (除非用户覆盖)
  └─ 未绑定 → 使用用户当前选择的模型
```

**用户覆盖机制**:
- 如果 `is_override_allowed: true`,用户可在对话中切换模型
- 如果 `is_override_allowed: false`,技能使用绑定模型,用户无法切换

#### 2.1.4 Admin UI 设计

**路径**: `/workspace/admin/models/skills` (已存在)

**新增 Tab**: "模型配置"

| 技能名称 | 绑定模型 | 允许覆盖 | 操作 |
|----------|----------|----------|------|
| image-generation | dall-e-3 | ✅ | [修改] [解绑] |
| deep-research | gpt-4o | ❌ | [修改] [解绑] |
| data-analysis | - | - | [绑定模型] |

### 2.2 管理员技能权限管理

#### 2.2.1 技能来源标记

**技能来源类型**:
| 来源 | 说明 | 权限 |
|------|------|------|
| `system` | 系统内置,不可更改 | 只能查看 |
| `admin` | 管理员安装 | 管理员可删除,用户只能启用/禁用 |
| `user` | 普通用户安装 | 创建者可删除 |

**数据模型**:
```python
class SkillMetadata(BaseModel):
    """技能元数据"""
    name: str
    source: Literal["system", "admin", "user"]
    installed_by: str | None  # user_id
    created_at: datetime
    updated_at: datetime
```

**存储**: 扩展 `SkillResponse`:
```python
class SkillResponse(BaseModel):
    # ... 现有字段 ...
    source: str  # 新增: system | admin | user
    installed_by: str | None  # 新增: user_id
```

#### 2.2.2 权限控制规则

| 角色 | system 技能 | admin 技能 | user 技能 |
|------|-------------|-------------|-----------|
| **成员** | 查看 | 启用/禁用 | CRUD (自己的) |
| **管理员** | 查看 | CRUD | CRUD (所有) |

**UI 展示**:
- 技能列表显示来源图标
  - 🛡️ system (灰色,只读)
  - 🔧 admin (蓝色)
  - 👤 user (绿色)
- 删除按钮: 只有权限足够的用户可见

#### 2.2.3 删除权限控制

**API 权限校验**:
```python
async def delete_skill(skill_name: str, user: AuthUser):
    skill = get_skill(skill_name)

    if skill.source == "system":
        raise HTTPException(403, "系统内置技能无法删除")

    if skill.source == "admin" and not user.is_owner:
        raise HTTPException(403, "管理员技能仅管理员可删除")

    if skill.source == "user" and skill.installed_by != user.id and not user.is_owner:
        raise HTTPException(403, "只能删除自己安装的技能")

    # 执行删除
    ...
```

### 2.3 飞书消息读取

#### 2.3.1 能力概述

基于飞书 `im.v1` API,实现消息读取能力:

| 功能 | API | 说明 |
|------|-----|------|
| 获取消息 | `GET /im/v1/messages/:message_id` | 获取单条消息详情 |
| 搜索消息 | `POST /im/v1/messages/search` | 搜索消息内容 |
| 获取会话历史 | `GET /im/v1/messages` | 获取群聊/单聊历史 |

#### 2.3.2 工具设计

**工具名称**: `feishu_message_search`

**输入参数**:
```python
class FeishuMessageSearchInput(BaseModel):
    """飞书消息搜索输入"""
    query: str = Field(..., description="搜索关键词")
    chat_id: str | None = Field(None, description="指定会话 ID,不填则搜索所有")
    start_time: str | None = Field(None, description="开始时间 (ISO8601)")
    end_time: str | None = Field(None, description="结束时间 (ISO8601)")
    limit: int = Field(default=20, description="返回消息数量")
```

**输出格式**:
```python
class FeishuMessageSearchResult(BaseModel):
    """飞书消息搜索结果"""
    messages: list[FeishuMessage]
    total: int
    has_more: bool

class FeishuMessage(BaseModel):
    """飞书消息"""
    message_id: str
    chat_id: str
    chat_type: str  # p2p | group
    sender: dict
    content: dict  # 消息内容 JSON
    create_time: str
```

#### 2.3.3 工具注册

**注册位置**: `deerflow/tools/community/feishu_tools.py` (新建)

**注册到 Agent**: `im` 工具组

#### 2.3.4 权限控制

- 只有配置了飞书渠道且启用的用户才能使用
- 需要 `im:message:read` 权限范围

## 六、非功能需求

### 6.1 安全性

- 技能删除权限校验必须严格
- 飞书 API 调用需要验证 token 有效性
- 防止通过 prompt injection 绕过权限

### 6.2 兼容性

- 现有技能不受影响
- 未绑定模型的技能使用原有逻辑
- 向后兼容 `extensions_config.json`

### 6.3 性能

- 模型绑定查询: < 10ms
- 飞书消息搜索: 利用飞书服务端搜索
