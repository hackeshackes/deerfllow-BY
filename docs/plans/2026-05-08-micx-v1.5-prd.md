## 一、版本目标与核心价值

### 1.1 版本愿景

v1.5 将 MicX 从**基础协作平台**进化为**高效企业协作与自动化平台**：

| 现有能力 | v1.5 新增 | 价值跃升 |
|---------|----------|---------|
| 任务/知识库仅基础分享 | **条件分支 + 循环/迭代** | 从"定时执行"到"智能自动化" |
| 任务执行结果仅创建者可见 | **工作区共享 + 实时状态** | 从"个人结果"到"团队资产" |
| 自动化结果无追踪 | **执行历史 + 状态监控** | 从"黑盒执行"到"透明可控" |
| 工作流仅技能调用 | **变量传递 + 条件判断** | 从"单步调用"到"流程编排" |

### 1.2 核心场景

**场景A - 智能自动化**
```
用户: 创建自动化 "每周一生成销售报告"
用户: 设置触发: Cron "0 9 * * 1"
用户: 添加步骤1: 调用 AI 生成报告
用户: 添加步骤2: IF 报告长度 > 1000字 THEN 压缩内容
用户: 添加步骤3: 发送邮件到团队
用户: 添加步骤4: FOR EACH 团队成员: 发送飞书通知
系统: 自动执行完整流程
```

**场景B - 团队协作自动化**
```
管理员: 在工作区A创建自动化 "新用户入职流程"
管理员: 设置: 新用户加入工作区时自动触发
管理员: 添加步骤: 发送欢迎消息 → 创建个人知识库 → 分配任务
工作区成员: 收到入职自动化通知和任务
管理员: 可查看工作区内所有自动化执行状态
```

**场景C - 条件触发工作流**
```
用户: 创建自动化 "客户反馈处理"
触发: 收到包含"投诉"关键词的消息
步骤1: AI 分析反馈内容
步骤2: IF 情绪负面 THEN 升级到管理员 + 发送确认邮件
步骤3: ELIF 情绪中性 THEN 创建任务跟踪
步骤4: ELSE 感谢用户 + 归档
```

**场景D - 循环批量处理**
```
用户: 创建自动化 "批量文档处理"
触发: 定时每天早上9点
用户: 设置要处理的文档列表（知识库查询结果）
FOR EACH 文档 IN 列表:
    - 提取关键信息
    - 更新到目标表格
    - 记录处理结果
发送汇总报告
```

### 2.1 自动化工作流 2.0 - 核心引擎

#### 2.1.1 工作流架构

```
┌─────────────────────────────────────────────────────────────┐
│                     Workflow Engine                          │
├─────────────────────────────────────────────────────────────┤
│  Trigger Layer          │  Logic Layer      │  Action Layer │
│  ───────────────────     │  ───────────      │  ────────────  │
│  • Cron 定时            │  • IF/ELSE 条件   │  • AI 调用    │
│  • 事件触发 (IM消息)    │  • FOR/EACH 循环  │  • API 调用   │
│  • Webhook 触发         │  • 变量赋值       │  • 消息发送   │
│  • 手动触发             │  • 函数计算       │  • 文档操作   │
│                         │  • 等待/延迟      │  • 任务创建   │
└─────────────────────────────────────────────────────────────┘
         ↓                      ↓                    ↓
    [Execution Context]    [Variables]        [Results]
```

#### 2.1.2 工作流定义数据模型

```python
class WorkflowStep(BaseModel):
    id: str                      # 步骤唯一ID
    type: str                    # 'ai_call' | 'condition' | 'loop' | 'action' | 'delay'
    name: str                    # 步骤名称
    config: dict                 # 步骤配置
    next_step_id: str | None     # 下一个步骤 (条件为true时)
    else_step_id: str | None     # 否则步骤 (仅condition类型)
    iteration_var: str | None   # 循环变量名 (仅loop类型)

class Workflow(BaseModel):
    id: str
    name: str
    description: str

    # 触发配置
    trigger_type: str            # 'cron' | 'event' | 'webhook' | 'manual'
    trigger_config: dict         # 触发器特定配置

    # 步骤定义
    steps: list[WorkflowStep]     # 有序步骤列表
    entry_step_id: str           # 入口步骤ID

    # 变量定义
    variables: list[Variable]    # 工作流级变量

    # 元数据
    workspace_id: str
    created_by: str
    created_at: datetime
    updated_at: datetime

class Variable(BaseModel):
    name: str
    type: str                    # 'string' | 'number' | 'boolean' | 'array' | 'object'
    default_value: Any | None
    description: str | None
```

#### 2.1.3 触发器类型

| 触发类型 | 配置字段 | 说明 |
|---------|---------|------|
| `cron` | expression, timezone | 标准 Cron 表达式，支持时区 |
| `event` | event_type, filter | 事件类型：im_message, user_joined, kb_updated |
| `webhook` | method, path | HTTP Webhook 触发，支持认证 |
| `manual` | - | 手动触发，可通过API或UI |

**Cron 配置示例**:
```yaml
trigger:
  type: cron
  expression: "0 9 * * 1"        # 每周一上午9点
  timezone: "Asia/Shanghai"
```

**事件触发配置示例**:
```yaml
trigger:
  type: event
  event_type: im_message
  filter:
    channel: "feishu"
    keywords: ["投诉", "反馈", "问题"]
```

**Webhook 配置示例**:
```yaml
trigger:
  type: webhook
  method: "POST"
  path: "/webhook/trigger"
  auth:
    type: "bearer"              # bearer token 认证
    token: "$WEBHOOK_SECRET"
```

#### 2.1.4 步骤类型详解

**1. AI 调用步骤 (ai_call)**
```yaml
type: ai_call
name: "分析客户反馈"
config:
  prompt: "分析以下反馈内容，提取关键信息：{{trigger.content}}"
  model: "gpt-4o"
  output_variable: "analysis_result"    # 结果存储变量
  timeout: 60
```

**2. 条件分支步骤 (condition)**
```yaml
type: condition
name: "情绪判断"
config:
  expression: "{{analysis_result.sentiment}} == 'negative'"
next_step_id: "step_escalate"           # true 时执行
else_step_id: "step_normal"             # false 时执行
```

**3. 循环迭代步骤 (loop)**
```yaml
type: loop
name: "批量处理文档"
config:
  iterable: "{{documents}}"            # 数组变量
  iteration_variable: "doc"            # 当前元素
  max_iterations: 100
next_step_id: "step_process_doc"
```

**4. 延迟步骤 (delay)**
```yaml
type: delay
name: "等待5分钟"
config:
  duration: 300                        # 秒
```

**5. 动作步骤 (action)**
```yaml
type: action
name: "发送邮件"
config:
  action_type: "send_email"
  to: "{{recipient}}"
  subject: "自动报告"
  body: "处理完成，结果：{{result}}"
```

#### 2.1.5 变量与表达式

**可用变量**:
- `trigger.*` - 触发器相关数据
- `steps.{step_id}.output` - 前置步骤输出
- `workflow.variables.*` - 工作流定义变量
- `context.*` - 运行时上下文 (user, workspace, timestamp)

**表达式示例**:
```
{{trigger.content}}
{{steps.step_1.output.summary}}
{{workflow.variables.max_retries}}
{{context.user.name}}
```

**条件表达式**:
```
# 字符串比较
{{trigger.type}} == "complaint"

# 数值比较
{{steps.count.value}} > 10

# 逻辑运算
{{is_urgent}} == true AND {{severity}} > 3
```

#### 2.2.4 循环执行机制

**循环执行流程**:
```python
async def execute_loop(context, loop_step):
    iterable = resolve_variable(context, loop_step.iterable)
    results = []

    for index, item in enumerate(iterable):
        if index >= loop_step.max_iterations:
            break

        # 设置循环变量
        context.variables[loop_step.iteration_var] = item
        context.variables['loop.index'] = index

        try:
            result = await execute_step(context, loop_step.next_step_id)
            results.append({'index': index, 'result': result, 'success': True})
        except Exception as e:
            if loop_step.continue_on_error:
                results.append({'index': index, 'error': str(e), 'success': False})
                continue
            raise

    context.step_outputs[loop_step.id] = results
```

**条件评估**:
```python
def evaluate_condition(expression: str, context: dict) -> bool:
    # 支持的运算符: ==, !=, >, <, >=, <=, AND, OR, NOT
    # 变量替换: {{variable.path}}
    # 返回布尔值
    pass

async def execute_condition(context, condition_step):
    result = evaluate_condition(condition_step.expression, context.variables)

    if result:
        next_step = condition_step.next_step_id
    else:
        next_step = condition_step.else_step_id

    if next_step:
```

### 3.3 定时调度集成

#### 3.3.1 APScheduler 集成

复用现有的 `ScheduledTask` 调度系统:

```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

class WorkflowScheduler:
    def __init__(self):
        self.scheduler = AsyncIOScheduler()

    async def schedule_cron_workflow(self, workflow: Workflow):
        trigger = CronTrigger(
            expression=workflow.trigger_config['expression'],
            timezone=workflow.trigger_config.get('timezone', 'UTC')
        )
        self.scheduler.add_job(
            self.trigger_workflow,
            trigger=trigger,
            args=[workflow.id],
            id=f"workflow_{workflow.id}",
            replace_existing=True
        )

    async def unschedule_workflow(self, workflow_id: str):
        job_id = f"workflow_{workflow_id}"
        if self.scheduler.get_job(job_id):
            self.scheduler.remove_job(job_id)
```
