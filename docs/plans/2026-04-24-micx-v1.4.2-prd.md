# MicX v1.4.2 PRD - 飞书全能力集成

**版本目标**: *基于 lark-oapi SDK，将飞书开放平台的全部核心能力引入 MicX Agent*

**日期**: 2026-04-24

---

## 一、背景

飞书开放平台提供 **2500+ 服务端 API**，涵盖消息、文档、多维表格、日历、任务、审批、邮件、通讯录等核心模块。v1.4.2 将把这些能力全部引入 MicX，让 Agent 能够在对话中直接操作飞书。

**技术选择**: `lark-oapi` Python SDK（B 方案，与 v1.4.1 一致），统一 SDK 版本，统一工具封装模式。

---

## 五、工具命名规范

所有飞书工具统一前缀 `feishu_`，命名格式：

```
feishu_{模块}_{操作}
```

示例：
- `feishu_doc_read` - 读取文档
- `feishu_bitable_write` - 写入多维表格
- `feishu_calendar_event_create` - 创建日程
- `feishu_approval_action` - 审批操作

## 六、权限控制

1. **飞书渠道必须启用**：`channels.feishu.enabled: true` 才可使用飞书工具
2. **工具级别权限**：某些高危操作（如发送消息、审批、邮件发送）需要 owner 角色
3. **飞书应用权限**：工具能访问的范围受飞书应用权限范围限制（用户在飞书中的文档/日历等访问权限）

## 七、用户使用流程（示例）

### 场景 1: 读取飞书文档

```
用户: "看一下这个飞书文档 https://xxx.feishu.cn/docx/Bsxxxxxx"
Agent: feishu_doc_read(document_id="Bsxxxxxx")
     → 返回文档内容
```

### 场景 2: 操作多维表格

```
用户: "帮我查一下产品需求库里状态是'进行中'的记录"
Agent: feishu_bitable_read(bitable_id="xxx", filter="状态='进行中'")
     → 返回记录列表
```

### 场景 3: 创建日程

```
用户: "明天下午2点帮我创建一个会议"
Agent: feishu_calendar_event_create(start_time="...", end_time="...", title="...")
     → 日程创建成功
```

### 场景 4: 查询用户

```
用户: "飞书上张三的邮箱是什么？"
Agent: feishu_contact_user(user_id_or_email="zhangsan")
     → 返回用户信息
```

### 场景 5: 发送邮件

```
用户: "帮我发一封邮件给 zhangsan@company.com"
Agent: feishu_mail_send(to=["zhangsan@company.com"], subject="...", content="...")
     → 邮件发送成功
```

### 场景 6: 审批操作

```
用户: "帮我看一下有没有待我审批的单子"
Agent: feishu_approval_list(user_id_type="open_id", approval_name="审批")
     → 返回待审批列表
```

## 九、风险点

1. **飞书权限范围**：工具能访问的内容受飞书应用权限和用户权限双重限制
2. **API 限流**：大量调用需处理限流错误，添加重试逻辑
3. **字段类型复杂性**：多维表格、任务等字段类型多样，需要处理不同类型序列化
4. **token 管理**：确保 tenant_access_token 有效期内处理，SDK 内置 token 管理
