## 四、技术优化

### 4.1 Docker 镜像优化 🛠️

**问题**: Docker 镜像体积 4.5G，远大于预期

**根因分析**:
1. dev target 包含完整编译工具链 (build-essential, nodejs, docker CLI)
2. 多阶段构建未正确使用 production target
3. 可能存在缓存问题

**优化方案**:

#### 方案 A: 确保使用 production target (推荐)
```yaml
# docker-compose-dev.yaml
langgraph:
  build:
    context: ./backend
    dockerfile: Dockerfile
    target: runtime  # 使用 runtime 而非 dev
```

#### 方案 B: 分离开发/生产 Docker 配置
- 开发: 使用 dev target，保留工具链
- 生产: 使用 runtime target，精简体积

**目标**: 将镜像体积从 4.5G 压缩到 2G 以下
