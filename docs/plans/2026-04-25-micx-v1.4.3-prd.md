## 二、对话标签

### 2.1 功能概述

在对话（Thread）上打标签，支持：

- 每个对话可打多个标签
- 标签支持自定义颜色
- 按标签筛选对话列表
- 标签管理（创建/编辑/删除）

### 2.2 数据模型

```python
# 标签表
class Tag(BaseModel):
    id: int
    workspace_id: str
    name: str
    color: str  # hex 颜色，如 "#FF6B6B"
    created_at: datetime
    created_by: str

# 对话-标签关联表
class ThreadTag(BaseModel):
    id: int
    thread_id: str
    tag_id: int
    created_at: datetime
```

**数据库迁移**:
```sql
CREATE TABLE tag (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    workspace_id TEXT NOT NULL,
    name TEXT NOT NULL,
    color TEXT NOT NULL DEFAULT '#6B7280',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by TEXT NOT NULL
);

CREATE TABLE thread_tag (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    thread_id TEXT NOT NULL,
    tag_id INTEGER NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (tag_id) REFERENCES tag(id) ON DELETE CASCADE
);

CREATE INDEX idx_thread_tag_thread_id ON thread_tag(thread_id);
CREATE INDEX idx_thread_tag_tag_id ON thread_tag(tag_id);
CREATE UNIQUE INDEX idx_thread_tag_unique ON thread_tag(thread_id, tag_id);
```

### 2.3 API 端点

#### 标签管理

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/tags` | 列出当前工作区所有标签 |
| `POST` | `/api/tags` | 创建新标签 |
| `PUT` | `/api/tags/{id}` | 更新标签（名称/颜色） |
| `DELETE` | `/api/tags/{id}` | 删除标签 |

**请求/响应示例**:

```bash
# 创建标签
POST /api/tags
{
  "name": "项目A",
  "color": "#FF6B6B"
}
→ 201
{
  "id": 1,
  "name": "项目A",
  "color": "#FF6B6B",
  "created_at": "2026-04-25T10:00:00Z"
}

# 列出标签
GET /api/tags?workspace_id=ws_xxx
→ 200
{
  "tags": [
    {"id": 1, "name": "项目A", "color": "#FF6B6B", "count": 5},
    {"id": 2, "name": "重要", "color": "#F59E0B", "count": 3}
  ]
}
```

#### 对话标签关联

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/threads/{id}/tags` | 获取对话的所有标签 |
| `POST` | `/api/threads/{id}/tags` | 为对话添加标签 |
| `DELETE` | `/api/threads/{id}/tags/{tag_id}` | 移除对话的标签 |

```bash

```

#### 按标签搜索对话

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/threads?tag_id=1` | 仅返回带有指定标签的对话 |
| `GET` | `/api/threads?tag_id=1,2` | 返回同时带有多个标签的对话 |

## 三、搜索增强

### 3.1 功能概述

增强对话搜索能力，支持：

| 能力 | 说明 |
|------|------|
| 标题检索 | 现有能力，按标题关键词匹配 |
| 全文检索 | 检索对话中的消息内容 |
| 多维度过滤 | 标签、工作区、用户、时间范围 |
| 结果排序 | 相关度 / 最新优先 |

### 3.2 API 端点

```bash
# 增强搜索
GET /api/threads/search?q=关键词&workspace_id=x&tag_ids=1,2&user_id=y&start=2026-01-01&end=2026-12-31&page=1&size=20&sort=relevance
```

**参数说明**:

| 参数 | 类型 | 说明 |
|------|------|------|
| `q` | string | 搜索关键词（可选，不传则做过滤） |
| `workspace_id` | string | 限定工作区 |
| `tag_ids` | string | 逗号分隔的标签 ID（AND 逻辑） |
| `user_id` | string | 限定创建者 |
| `start` | date | 创建时间起 |
| `end` | date | 创建时间止 |
| `page` | int | 页码，默认 1 |
| `size` | int | 每页数量，默认 20，最大 100 |
| `sort` | string | `relevance`（相关度）/ `latest`（最新优先） |

**响应**:

```json
{
  "threads": [
    {
      "id": "thread_xxx",
      "title": "AI 每日报告 - 2026-04-25",
      "workspace_id": "ws_xxx",
      "user_id": "user_xxx",
      "tags": [{"id": 1, "name": "项目A", "color": "#FF6B6B"}],
      "message_count": 42,
      "created_at": "2026-04-25T08:00:00Z",
      "updated_at": "2026-04-25T10:30:00Z",
      "snippet": "...匹配到的消息内容片段...",
      "relevance_score": 0.95
    }
  ],
  "total": 15,
  "page": 1,
  "size": 20
}
```

### 3.3 实现方案

**标题检索**: 现有 SQLite `LIKE` 查询

**全文检索**: SQLite `LIKE` 遍历 messages 表

```sql
SELECT DISTINCT t.* FROM thread t
JOIN message m ON m.thread_id = t.id
WHERE m.content LIKE '%关键词%'
   OR t.title LIKE '%关键词%'
ORDER BY t.updated_at DESC
LIMIT 20 OFFSET 0;
```

**索引优化**:
```sql
CREATE INDEX IF NOT EXISTS idx_thread_title ON thread(title);
CREATE INDEX IF NOT EXISTS idx_message_content ON message(content);
CREATE INDEX IF NOT EXISTS idx_thread_workspace ON thread(workspace_id);
CREATE INDEX IF NOT EXISTS idx_thread_user ON thread(user_id);
CREATE INDEX IF NOT EXISTS idx_thread_created ON thread(created_at);
```

## 六、风险点

1. **全文检索性能**：大对话表 + LIKE 查询可能慢，上线后监控查询延迟
2. **标签数据迁移**：thread 表无标签字段，通过关联表实现，兼容性好
3. **搜索索引**：SQLite 在数据量大时全文检索不如专业搜索引擎，v1.4.3 先用 LIKE，后续可升级 PostgreSQL 全文检索
