# MicX v1.4.3 PRD - 技能可见化 / Feishu 对话管理 / 知识库分层

**版本目标**: *技能全局可见 + Feishu 对话纳入管理 + 知识库分层管理*

**日期**: 2026-04-25

---

## 一、背景

v1.4.2 完成了飞书平台 57 个工具的建设。v1.4.3 聚焦三个长期困扰用户体验的问题：

1. **技能不可见**：大量技能因无 share 记录，普通用户完全看不到，无法使用
2. **Feishu 对话不可管**：通过飞书机器人发起的对话，在 Web UI 中对管理员也不可见，无法管理
3. **知识库无分层**：普通用户和管理员创建的知识库无法区分，所有人创建的都是个人知识库

---

## 二、问题详细分析

### 2.1 技能可见性问题

**当前逻辑** (`get_visible_skills_for_user`):

```python
# user_skill_store.py line 242-279
def get_visible_skills_for_user(user_id, is_owner, workspace_id=None):
    if is_owner:
        return [s.name for s in load_skills(enabled_only=False)]  # admin 看全部

    shares = _get_skill_shares()
    for skill in all_skills:
        share = shares.get(skill.name)
        if share is None:
            continue  # ← 无 share 记录 = 对普通用户完全不可见
        elif share.owner_id == user_id: ...
        elif share.visibility == "public": ...
        elif share.visibility == "workspace": ...
```

**问题根因**:
- `skill_shares.json` 中没有记录的技能，普通用户 `continue` 跳过了
- 结果：大量内置技能（system: 前缀）和未主动分享的技能对普通用户不可见
- admin 手动 share 太繁琐，不现实

**期望行为**:
- 所有技能对所有用户可见（只需 enable/disable）
- 普通用户：可开关任意技能（对自己生效），不可删除
- 普通用户：对自己创建的技能拥有完整管理权（编辑、删除、分享）
- admin：拥有所有技能的完整管理权

### 2.2 Feishu 对话不可见问题

**当前逻辑**:

```
Feishu 用户发消息
    → lark-oapi WebSocket 收到事件
    → manager._create_thread() 创建 LangGraph thread
    → channel_store.json 存储 {chat_id, topic_id} → thread_id 映射
    → thread 在 LangGraph Server，无 workspace_id，无 owner_id
```

**问题根因**:
- Feishu 线程创建时没有关联 MicX 用户（`user_id` 是飞书用户的 open_id，不是 MicX user_id）
- 线程在 LangGraph Server 创建，但 ownership metadata 没有写入
- Web UI 线程列表基于 `workspace_id` + `user_id` 过滤，Feishu 线程天然缺失这两个字段

**期望行为**:
- admin 可以看到所有线程（包括 Feishu 发起的）
- admin 可以将 Feishu 对话共享到指定工作区
- admin 可以管理（删除、修改可见性）任意线程

### 2.3 知识库无分层问题

**当前逻辑** (`knowledge.py`):

```python
# 任何登录用户都能创建，创建者即 owner
conn.execute(
    """INSERT INTO knowledge_bases (id, user_id, workspace_id, visibility, ...)
    VALUES (?, ?, ?, 'private', ...)""",
    (kb_id, user.id, workspace_id, ...)
)

# 只有 owner 能删除
if not row or row["user_id"] != user.id:
    raise HTTPException(status_code=404, detail="...")
```

**问题根因**:
- 所有知识库都是 `visibility='private'`，无全局概念
- 没有 admin 全局知识库一说
- admin 无法为所有用户预置公共知识库

**期望行为**:
- 普通用户创建的知识库：仅用户个人 + 可分享给工作区
- admin 创建的知识库：可标记为 `global`（全局可见，所有人可检索但仅 admin 可编辑/删除）
- admin 管理后台增加知识库管理页面（放在 IM 渠道下方）
- 用户侧知识库（`/api/knowledge`）位置保持不变

## 三、解决方案

### 3.1 技能可见化

**修改 `get_visible_skills_for_user`**:

```python
def get_visible_skills_for_user(user_id, is_owner, workspace_id=None):
    if is_owner:
        return [s.name for s in load_skills(enabled_only=False)]  # admin 看全部

    shares = _get_skill_shares()
    visible = []
    for skill in all_skills:
        # 新逻辑：无 share 记录的技能 → 默认可见（public）
        share = shares.get(skill.name)
        if share is None:
            visible.append(skill.name)  # 默认公开
        elif share.owner_id == user_id:
            visible.append(skill.name)  # 自己的
        elif share.visibility == "public":
            visible.append(skill.name)
        elif share.visibility == "workspace" and user_is_in_workspace(user_id, share.workspace_id):
            visible.append(skill.name)
    return visible
```

**修改删除权限** (`skills.py`):

```python
# 当前：需要是 owner 才能删除
# 修改后：
# - admin 可以删除任何技能
# - 普通用户可以删除自己创建的技能（share.owner_id == user.id）
# - system: 前缀的技能任何人不得删除
```

**新增技能开关状态存储**:
- 使用已有的 `extensions_config.json` 的 `skills[skill_name].enabled`
- 为每个用户存储独立的 enabled 状态（`user_skill_config.json`）

### 3.2 Feishu 对话可见化

**方案：为 Feishu 线程建立 MicX 用户关联**

在 channel store 中增加 `micx_user_id` 和 `micx_workspace_id` 字段：

```python
# channel_store.json
{
    "feishu:chat123":
        "thread_id": "thread_xxx",
        "user_id": "feishu_open_id",      # 飞书用户
        "micx_user_id": "micx_user_123",   # 新增：关联的 MicX 用户
        "micx_workspace_id": "ws_default", # 新增：归属工作区
        "created_at": 1700000000.0,
        "updated_at": 1700000000.0
}
```

**两种关联方式**:
1. **自动关联**：Feishu 用户首次发消息时，自动关联到对应 MicX 用户（通过飞书 open_id 匹配）
2. **手动关联**：admin 在后台手动将 Feishu chat 分配给 MicX 用户/工作区

**修改线程列表 API**:
- Admin 请求：返回所有线程（`is_owner=True` 时不过滤）
- 普通用户：仅返回自己创建或 workspace 共享的线程

**新增 Admin 页面** (`/workspace/admin/conversations`):
- 列出所有 Feishu/Slack/Telegram 会话
- 显示关联的 MicX 用户和工作区
- 支持手动关联/分配
- 支持删除、修改可见性

### 3.3 知识库分层

**数据模型变更**:

```sql
-- 知识库表新增 is_global 字段
ALTER TABLE knowledge_bases ADD COLUMN is_global BOOLEAN DEFAULT FALSE;

-- 新增 is_global 索引（性能优化）
CREATE INDEX IF NOT EXISTS idx_kb_is_global ON knowledge_bases(is_global);
```

**API 变更**:

| 端点 | 变更 |
|------|------|
| `POST /api/knowledge` | 普通用户：自动设置 `is_global=False`；admin：可选 `is_global=True` |
| `GET /api/knowledge` | 返回：个人 + 共享给我的 + 全局（is_global=True） |
| `DELETE /api/knowledge/{id}` | 仅 owner 或 admin 可删除 |
| `PUT /api/knowledge/{id}` | 仅 owner 或 admin 可更新 |

**Admin 管理页面** (`/workspace/admin/knowledge`):
- 放在 IM 渠道配置下方（`/workspace/admin/models/mcp/channels` 同级）
- 功能：创建全局知识库、查看所有知识库、删除全局知识库

### 4.1 技能相关

| 端点 | 方法 | 变更 |
|------|------|------|
| `GET /api/skills` | GET | **修改**：普通用户可见所有技能（无需 share 记录） |
| `DELETE /api/skills/custom/{name}` | DELETE | **修改**：admin 可删除任意；普通用户可删除自己创建的 |

### 4.2 线程相关

|------|------|------|
| `GET /api/threads` | GET | **新增**：admin 返回所有线程（含 Feishu 发起的） |

### 4.3 知识库相关

| 端点 | 方法 | 变更 |
|------|------|------|
| `POST /api/knowledge` | POST | **修改**：admin 可创建 `is_global=True` 的知识库 |
| `GET /api/knowledge` | GET | **修改**：返回个人 + 共享 + 全局 |
| `DELETE /api/knowledge/{id}` | DELETE | **修改**：admin 可删除任意 |
| `GET /api/admin/knowledge` | GET | **新增**：admin 获取所有知识库（含全局） |

## 八、风险点

1. **技能可见化后性能**：所有技能可见 → `GET /api/skills` 返回量增大，但 skill 数量有限（<100），影响可控
2. **Feishu 用户自动关联**：需要维护飞书 open_id → MicX user_id 映射表，新增一张表或扩展 channel_store
3. **知识库 is_global 字段**：需要 ALTER TABLE，现有数据默认可正常访问
